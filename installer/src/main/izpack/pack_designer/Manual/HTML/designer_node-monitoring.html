Node Monitoring


Topic content

Description of the dialog fields for the node monitoring

Local name:                <name to specify a remote node, can be chosen freely>
Instancename:        <the value of the parameter "nodeid" in environment_settings.xml of the remote node, default value is "ORC">

Group:                the group name is used for including more nodes with the same instance name - group name and instance name together have to be unique
Host:                        <name or IP-address of the remote node>
Port:                        <the value of the parameter "servlet.port.http" in environment_settings.xml of the remote node, default value is "8019", is SSL ticked, the port from                                        "servlet.port.https" is used>
                 <group name="Http">
                         <parameter name="servlet.port.http" value="8019"/>
                         <parameter name="servlet.port.https" value="8443"/>

User/Pwd:                <a run time user from the remote node that has the role "Remote.Administrator">
SSL:                        <the port depends on the selection here!>

Advice

•The remote node has to have a runtime user with the role "Remote.Administrator" assigned. This user is deposited for logging in in this dialog. It is sufficient that the user has this role, it is not necessary that the user is administrator!

Even the user "admin" does not have this required role automatically!

•For controlling, what the user can see in the monitor from the remote nodes, the user that is currently logged in is handed over. This means that the user that is logged in has to be existent in the remote system. For example, if you log in with the user "Hugo1" at node 1 and wants to monitor node 2, on the instance of node 2 the user "Hugo1" has to exist, too.
