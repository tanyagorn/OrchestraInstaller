How to work on a branch or a tag


Topic content

Basically the same as described in the section on Working with branches and tags.

In Git you cannot work on a tag directly. In Git a tag is just a marker for a revision of your repository.

In Subversion  a tag or a branch is a copy of the data to the branch or tag. There is no technical difference in Subversion.

When you want to switch to a tag, a dialog opens asking you the name of a branch. Orchestra automatically creates a branch from the tag so you can work on it like you can do it in Subversion.
