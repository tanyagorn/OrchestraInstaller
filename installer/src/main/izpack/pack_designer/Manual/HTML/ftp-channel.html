FTP channel


Topic content

Description

A FTP Channel is an adapter to read, write files and execute FTP commands on a FTP server. FTP, SFTP, FTPs are supported by this channel. If you would like to use a FTP – Channel, you will necessarily require a user with which you have the access rights to the respective FTP-Server. In order to create a user within Orchestra please refer to Credentials reference manual.

Creation

To create a FTP Channel follow the steps described in the general description of Channels

In versions prior 4.6.0.0, if you use a security credential with technical login with username/password authentication, not the account name was used but the alias of the credential. So the credential name has to be the name of the SFTP user!

Step by Step

•Server entry - Environment entry for the ftp connection. For more information about creation of this environment entry please refer to FTP Connection topic.

Use of FTP Commands and Stream Filter

FTP Commands

The table below show the available commands supported by the FTP Channel.

Command

Description

Change_EXT_UNIQUE

Change the file suffix and ensure that it is a unique name (by adding a number to the name, for ex. File42.txt instead of File.txt)

GET

Read a file from an FTP server

MGET

Read multiple files as MessageList from the FTP server by file pattern

MKDIR

Create a folder on a FTP server

MOVE

Move a file to another folder

MOVE_CHANGE_EXT

Move a removed file to another folder and ensure that the file name is unique (by adding a number to the name, for example, File42.txt instead of File.txt)

PUT

Create a file on an FTP server

REMOVE

Delete a file on an FTP server

RENAME

Rename a file on an FTP server

RENAME_UNIQUE

Rename a file on an FTP server and ensure that the name is unique (by adding a number to the name, for example, File42.txt instead of File.txt)

RMDIR

Delete a folder on an FTP server

 

Insert FTP Command

To include FTP commands in the FTP channel, please:

1.Open the FTP channel for which you would like to insert a FTP command

2.Choose the entry [add]

3.The dialogue [Add new FTP Command] will appear. Choose your desired ftp command then click on [next].

FTP Channel – Add Command

4.configure your selected ftp command and click on [finish] to save your configuration

FTP Channel – Add Command

Using Stream Filter in FTP Channel

In FTP channel, the use of the Stream-Filter is possible as well; however, the approach for using Stream-Filter here differs from the approach in other channels.

The Stream Filters are possible for the following FTP commands:

•GET

•MGET

•PUT

To use a Stream-Filter in the FTP channel, please:

1.Open the FTP channel for which you would like to insert a FTP command

2.Choose the FTP command for which you would like to define the Stream-Filter, and click on [Edit].

FTP Channel – Command Edit

3.The dialogue [Edit FTP Command] will appear. Here you can configure the Stream-Filter as usual. Finally, click on [OK].

FTP Channel – Command – Stream Filter

JMS message types

Depending on the linked JMS message type different input messages are generated.

Parameter

Description

TextMessage

Depending on the deserializer used a Orchestra Message from the incoming JMS message is generated.

BytesMessage

Depending on the deserializer used a Orchestra Message from the incoming JMS message is generated.

StreamMessage

From the incoming JMS message an Orchestra Message is generated. The incoming stream is contained in a field <value>.

MapMessage

Generates an Orchestra message containing all the key value pairs of the incoming JMS message <messagexyz> <propertyname1>value1</propertyname1> <propertyname2>value2</propertyname2> </messagexyz>

ObjectMessage

The incoming JMS message generates an Orchestra message without any content. Additionally, an input object of type "any" is generated which contains the incoming object (must be cast to use it)

 

Testing the channel

You may test the FTP Channel like every other outbound channel.

When the channel is configured the user can test its functionality by clicking on the test symbol in the tool bar of the Designer.

Then a dialog opens showing the events the channel receives or an error message.
