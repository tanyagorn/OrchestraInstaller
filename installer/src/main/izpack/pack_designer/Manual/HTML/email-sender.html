Email Sender


Topic content

Description

An Email Sender provides functionality to send emails to a remote mail server.

Creation

To create an Email Sender channel follow the steps described in the general description of Channels

Configuration

The dialog to configure the Email Sender looks like:

base view of the channel email sender

Step by Step

•Server entry - The mail server configuration. The server connection must before be created as environment entry (Email server connection).

•User - Select the user to login to the e-mail server. This user has to be created in advance in the security settings as credential.

•Server URI - This value is generated based upon Service entry selection.

•Mode - Select the mode of the email message. Controls the number and type of arguments of the channel.

oSimple: The e-mail body is given as text with optional variable references contained. A variable reference looks like $(variable-name). The variable-name then appears as parameter of the channel.
Actually there are two variants of this mode: One with zero or one attachment and another with many attachments.
If you select zero or one attachment the channel will have two additional parameters ATTACHMENT and ATTACHMENTNAME. The parameter ATTACHMENT contains a message whose content is serialized as mail attachment, the parameter ATTACHMENTNAME contains the name of this attachment.
If you select many attachments the channel will have an additional parameter ATTACHMENTS containing a message list.

oComplex: The e-mail body is created by serializing a message. The channel has a parameter MSG containing the body message.
Actually there are three variants of this mode: One with zero or one attachment, a second with many attachments and another with HTML message, Text message, images, Attachments.

•FROM - E-Mail address of the sender.

•TO, CC, BCC - E-Mail address of the recipient.

Note: Variables can also be used. These will then be passed to the channel from the outside. For this purpose, you can simply put $(variable-name) in the field where a variable should be used.

•Subject - Subject of the e-mail.

•Mail body - Configuration section for the email body

omail body - An optional text to appear in the e-mail body (only for simple mode)

oDefault type - MIME type of the e-mail body. This value is also generated by selection serializer option and can be changed manually. For more detailed information about message types please refer to Message Types reference manual.

oSerializer - Configure the Serializer  to create the content from a message into body of the mail. For more information about serilizers please refer to Serializer reference manual.

•Attachement(s) - Configuration section for the email attachments.

oDefault type - MIME type of the e-mail attachment. This value is also generated by selection serializer option and can be changed manually. For more detailed information about message types please refer to Message Types reference manual.

oSerializer - Configure the Serializer  for attachments of the email. For more information about serilizers please refer to Serializer reference manual.

Testing the channel

You may test the Email Sender Channel like every other outbound channel.

When the channel is configured the user can test its functionality by clicking on the test symbol in the tool bar of the Designer.

Then a dialog opens showing the events the channel receives or an error message.
