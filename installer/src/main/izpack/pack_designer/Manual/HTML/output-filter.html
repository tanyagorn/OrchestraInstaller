Output filter


Topic content

Concept

When dealing with binary data, orchestra has to transfer the binary content (e.g. file) to the internal format of orchestra. In this context two concepts are important. This are Serializers and filters. A Serializier is responsible for converting an orchestra message to an external byte representation. After this transformation it may be necessary to transform the generated byte data. This post processing after a serializer is done by output filters and is organized as sequence of output filters that can be applied after the serializer. In this chapter we describe the creation of such a user defined filter.

The generated template works on the idea of a filter stream. The Serailizer writes its data to an output stream.  Depending on the configuration, within such a filter chain, the user defined filter will also be invoked. Whenever the user defined filter is invoked by the write method, it is allowed to change the written bytes, also new bytes can be added. In the above example, when write method is invoked, normally the original value is passed to the next, underlying output stream. Only if  a semicolon is written, it is replaced by a "/" sign.
