General node settings


Topic content

These settings have to be made for all types of Orchestra installations.

language

Defines the default language of the current Orchestra node.

mode

Defines the operation mode of the current Orchestra node.
STANDALONE: one Orchestra runtime is responsible for execution of the modeled scenarios, standard mode
CLUSTERNODE: all installations of one cluster use the same database in order to provide security if one installation crashes
 
It is possible to use the cell system in both operation modes!

nodeid

Defines the unique name of this node inside a cell. The unique identifier is the combination of the parameters cellname and nodeid. Only letters digits and '-' are allowed. The maximum length must not exceed 10 characters.

cellname

Name of the owning cell, has to be unique (two cells are not allowed to have the same cell name).

deploy.path

Obsolete: Temporary directory that is used during deployment of scenario elements.

maxMemoryUsage

Defines a threshold (in percent) for the memory consumption. When more memory than the given value is used, Orchestra forces different subsystems to store newly created objects into the database. This behavior finishes, when the memory situation relaxes. Allowed values are between 50 and 100. (Note: The maximum available memory is defined by the global virtual machine settings)
