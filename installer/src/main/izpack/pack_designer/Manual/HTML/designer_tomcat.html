Tomcat


Topic content

Here it is described how you have to configure the tomcat server if you want it to provide a HTTPS connection for the Orchestra monitor.

Configure the file <path to tomcat>/conf/server.xml in the following way. The red part is the important part. This connector has to be activated.

<Connector port="8080"

         protocol="HTTP/1.1"

         connectionTimeout="20000"

         redirectPort="8443" />
<!-- A "Connector" using the shared thread pool-->
<!--
<Connector executor="tomcatThreadPool"

         port="8080" protocol="HTTP/1.1"

         connectionTimeout="20000"

         redirectPort="8443" />
-->
<!-- Define a SSL HTTP/1.1 Connector on port 8443
This connector uses the JSSE configuration, when using APR, the connector should be using the OpenSSL style configuration described in the APR documentation -->
<Connector port="8443"

         protocol="HTTP/1.1"

         SSLEnabled="true"

         maxThreads="150"

         scheme="https"

         secure="true"

         clientAuth="false"

         sslProtocol="TLS"

         keystoreFile="<path to keystore>/orchestra.keystore_soffico"

         keystorePass="<password for keystore>" />

 

Add to the web.xml file in the end the following lines. The red part has to be inserted!

...

<security-constraint>    
  <web-resource-collection>        
    <web-resource-name>orchestra</web-resource-name>
        <url-pattern>/*</url-pattern>        
    <http-method>GET</http-method>        
    <http-method>POST</http-method>    
  </web-resource-collection>    
  <user-data-constraint>        
    <transport-guarantee>CONFIDENTIAL</transport-guarantee>    
  </user-data-constraint>
</security-constraint>    
 
<!-- Default page to serve -->
<welcome-file-list>
  <welcome-file>Orchestra_Web.html</welcome-file>
</welcome-file-list>
 
</web-app>
