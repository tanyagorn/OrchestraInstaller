REST Service Provider


Topic content

Description

The purpose of the REST Service Provider is to enable you to implement REST web services on your own, without a need for a specific programming language knowledge. Using REST Service Provider, Orchestra will act as an external REST web service which exposes different APIs based upon your requirements. The Rest Service Provider concept used for creating web service contains only the declaration or REST methods that the user wants to implement. The actual implementation is done by creating a process model object and assigning it to a particular REST method as its implementation

REST Provider in Orchestra

 

Orchestra provides you with two different ways of creating REST Service Provider object. You can create it manually or by importing WADL (Web Application Description Language) file definition.

Creating

In order to manually create a REST Service Provider the following steps needs to be performed:

1.Right Click on the REST Service Provider group from Scenario element tree within Orchestra designer.

2.Select the Create option to create a new REST Service Provider entry.
 

creating a REST service provider

3.In the opened dialog enter the Name and Description (optional) for your REST Service Provider and click on the Finish button.

name and describe the new REST service provider

4.The user will be presented with the configuration panel for the created REST service provider.
 

Importing a WADL file

 

WADL (Web Application Description Language) is an XML based language to describe REST web applications similar to WSDL which is used to describe SOAP web applications.

Orchestra provides you with the ability to create REST Service Providers by importing a WADL file definition.

1.Right Click on the REST Service Provider option from orchestra designer main menu.

2.Select Import WADL option to create a new REST Service Provider.

Importing a WADL file

3.You will be presented with the Import Rest service description dialog.

4.Select the WADL description file and click the Next button.

5.On the second screen name the REST Service and click on the Finish button.

6.After clicking the Finish button, a new REST Service Provider object will be created and the configuration panel will be opened.

Configuration

1.Right Click on the created REST Service Provider in designer menu and select the properties option in order to change name an description of your REST Service Provider

2.Double click on your REST Service Provider in the scenario elements tree to open the configuration panel.

3.Click on the Resources element of your REST Service Provider object.

basic configuration of the REST service provider

•Description of the REST service provider object.

•Authentication of the REST service provider object determines the type of authentication you want to use for your REST method calls. For more detailed information about this configuration refer to configuring authentication.

•Use HTTPS determines the usage of secure HTTP protocol. Transport protocol is by default set to HTTP.

•Base path value is a part of URL-Pattern of the web service calls and it is automatically generated in order to provide unique value. Check overwrite default path option in order to enter a custom base path value.

•Overwrite default path If you create a REST provider manually, the base path is generated automatically to ensure that the path is unique. You may set your own path if you know that it is unique by setting this option.

•Configuring serializers and deserializers used within the REST service:

With a REST service typically JSON or XML data are sent over the TCP stream. The specific serializer/deserializer to use, is selected by the Content-Type. Therefore the serializers are configured at the root element of a REST service, that is within the resources element.

Seralizers in the context of a REST service

Most important is probably the JSON writer and the JSON reader. Within the context of REST services you cannot select a specific message type in the JSON writer. Instead you select the entry <configured type>.

In this case the message type is used which is configured in the request or response of the REST service using the Serializer to write JSON data.

 

Note: On the left side of the image above, you can see the created elements of the REST Service provider. The parent node of the provider is Resources element which contains the URL information of your Rest service. Resource child element contains more detailed information about URL-Pattern of web service calls together with the list of implemented REST methods.

 

After applying configuration setting for the Resources element you will need to configure each resource element with corresponding methods. For more details see the sections Resource and Method.
