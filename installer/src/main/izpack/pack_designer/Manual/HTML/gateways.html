Gateways


Topic content

Description

In process models, it is often necessary to diverge or converge a process in other to control process flow. For this purpose we use gateways. Orchestra implements following simple types of gateways:

- OR fork

- OR join

- Exclusive OR fork

- Exclusive OR join

OR - fork and join gateway

OR fork gateway is used for splitting process flow in parallel executions, in case condition defined for the execution edge is satisfied.

OR join gateway simply joins two or more tasks in a process flow.

In Orchestra designer, OR fork gateway is shown with white color and OR join gateway is shown in gray color. Also, sign o is used to denote OR type of gateway.

In following process model, we are using OR fork gateway to split process and then OR join gateway to merge two diverged flows in one flow:

OR gateway

Here, callA and callB will be both executed, if conditions that are set on edges are satisfied.

In order to set condition, simply double click on the edge (between OR gateway and call activity) and you will be able to configure condition:

Gateway condition

Condition can be set by using XPath or Java code.

XOR - fork and join gateway

XOR fork gateway is used to split  business process to two or more mutually exclusive paths.

XOR join gateway, similarly to OR join, simply joins two or more flows into one flow.

In Orchestra designer, XOR fork gateway is shown with white color and OR join gateway is shown in gray color. Also, sign x is used to denote XOR type of gateway.

In following process model, we are using XOR fork gateway to split process and then XOR join gateway to merge two diverged flows in one flow:

XOR gateway

In shown example either callA or callB will be executed. The first one that fulfills condition set on edge will be executed. Condition configuration is same like in OR gateway.
