Request appointments by using Query Parameters (Method GET)


Topic content

 

In this chapter we will create a Orchestra REST Client for requesting open appointment-slots with doctor “rbauer”  and save the request to your File system. According to 3.3.5 Request appointments by using Query Parameters (Method GET) we request the appointmentService by using the URL

http://localhost:8080/AppointmentMgr/appointments/doctors/{doctorname}/slots

 

As an additional information for telling the server, which appointment slot  we want to book, we have to enhance the URL with the Query date=20162204&status=open of our Request.

 

Figure 1 – Request an appointment by using Query Parameters - REST-Service Method Request Configuration

 

 

1.Copy your shell scenario from chapter 4.5.1 Preparation.

2.Open your process model.

3.Add the Rest Service Client and a call activity to your process model.

4.Link all elements in your modell.

Figure 2 – Request an appointment by using Query Parameters - Process Modell

5.Now you can configure your call activity for the AppointmentMgr-Service.

6.Open the register Service method and select the GET-Method for /appointments/doctors/{doctorname}/slots.

7.Assign the Response MSG of the call to the CURMSG of the process modell.

8.Assign the Query-Parameters date “20162204”, status “open” and the Template-Parameter doctorname “rbauer”.

 

Figure 3 – Request an appointment – Call configuration

9.Start your Test-Engine, and have a look at your result-File.

 

File Content

<?xml version="1.0" encoding="UTF-8"?>

<openSlotList>

  <slots date="20162204" doctorName="rbauer" end="930" id="13" start="900"/>

</openSlotList>
