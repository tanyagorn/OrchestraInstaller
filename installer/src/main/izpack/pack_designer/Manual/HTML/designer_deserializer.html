Deserializer


Topic content

Concept

All Channels reading byte streams need a deserializer to parse the content of the stream and create a structured message from it.

Note that the terms Inbound channel and Outbound channel don't refer to the direction in which the data flow but to to initiator of the call. So an outbound channel might read data and therefore needs a deserializer (e.g. the File Reader ) as well as an inbound channel might write data (e.g. the HTTP GET Receiver) and therefore needs a serializer.

E.g. if you want to read a CSV file using the File Reader, you need a CSV deserializer to transform the content into a structured message, that is a DOM tree.

Creation

A deserializer is always created in the context of configuring a Stream channel.

 

Basically the user has to select a deserializer, select an appropriate message type (if necessary) and then click on the button marked with ... to configure the deserializer (if necessary).

If no deserializer is selected so far or only the binary reader is selected and the user selects a message type associated with a particular deserializer, then that deserializer is automatically selected too.

Below the line for selecting the deserializer, you will find a line where you can select a Stream filter by double clicking the arrow -->

List of input filters

This way you also may build a sequence of filters.

To configure a filter the user has to double click on the name in the sequence.
