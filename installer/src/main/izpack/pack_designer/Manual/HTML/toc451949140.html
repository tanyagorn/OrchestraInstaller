Configuration


Topic content

1.Right Click on the created REST Service Provider in designer menu and select properties option in order to change name an description of your REST Service Provider.

               

Figure 36 – REST Service Provider - edit properties dialog

2.Double click on your REST Service Provider under REST Service Provider option in designer menu, to open configuration panel.

3.Click on the Resources element of your REST Service Provider object.

Figure 37 – REST Service Provider - resources configuration panel

•Description of the REST service provider object.

•Authentication of the REST service provider object determines the type of authentication you want to use for your REST method calls. For more detailed information about this configuration refer to Credentials reference manual.

•Use HTTPS determines the usage of secure HTTP protocol. Transport protocol is by default set to HTTP.

•Base path value is a part of URL-Pattern of the web service calls and it is automatically generated in order to provide unique value. Check overwrite default path option and in order to enter custom base path value.

Note: On the left side of the image above, you can see the created elements the REST Service provider. The parent node of the provider is Resources element which contains the URL information of your Rest service. Resource child element contains more detailed information about URL-Pattern of web service calls together with the list of implemented REST methods.

4.After applying configuration setting for the Resources element you will need to configure each resource element with corresponding methods. For more details see the sections Resource and Method.
