Trusted Server and Trusted Signer

Trusted server and trusted signer are two mechanisms that are used for are public key infrastructure.

Some basics about certificates and servers:

•There are some root certificates from certification authorities that are self-signed. That means they are signed with the private key of the authority and contain the public key.

•Intermediate certificates are signed by certification authorities. They contain the public key of the intermediate certificate and are signed with a digital signature from the root certificate.

•This leads to certificate chains, each certificate is signed with the private key from the certificate that is in the chain above (except from the root certificate, which is self-signed).

•A server has a truststore. A truststore contains certificates from other parties that you expect to communicate with, or from Certificate Authorities that you trust to identify other parties.

•A server can also have a keystore. A keystore contains the private keys and the certificates with their corresponding public keys.
