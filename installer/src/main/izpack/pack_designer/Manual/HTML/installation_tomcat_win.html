Tomcat


Topic content

Description

Apache Tomcat is an open-source web-server and web-container, which implements the specifications for Java Servlets and JavaServer pages. It runs web applications which are based on Java.

 

Please Change the standard password of the tomcat administrator to an individual password given by your own. This is essential to prevent security attacks.

Please refer to chapter Firewall Settings. In this chapter the configuration for "https"-Only access to the tomcat server is described. This is essential to prevent security attacks.

Installation

We recommend Apache Tomcat 8 to run Orchestra. It can be downloaded from the following link http://tomcat.apache.org/download-80.cgi

Install the version for your operating system (32 or 64 bit). Follow the instructions of the installation assistant.

Choose normal installation.

 

Choose your used ports and service name. It is recommendable to set up an admin here.

 

Indicate to the installed JDK8 directory.

 

Click on install to complete the installation.

 

After the installation of tomcat,  special settings for the java virtual machine have to be done. Please open the Tomcat configuration dialog and switch to the register "Java".

 

Within the following dialog, additional classpath entries must be added.

Classpath settings

 

Add the following two path entries at the end of the standard classpath:

•<tomcat installation directory>\webapps\orchestra\WEB-INF\lib\orchestra-jre-url-extension.jar

•<tomcat installation directory>\webapps\orchestra\WEB-INF\lib\emds_logging.jar

 

Also the memory options should be adjusted.Please set the values to the following default values:

Initial memory pool: 1024 (=1GB)

Max memory pool: 2048 (=2GB)

 

'Initial memory pool' stands for the initial RAM at the start of Tomcat. Max memory is the 'maximum amount of memory that java can allocate.

When the memory consumption gets increases memory is allocated up to max memory pool.

 

 

Furthermore,  additional Java Options have to be defined. This is also done the Tomcat Configuration Application.

Java options

 

With the java options the runtime behavior of the java virtual is adjusted. Please add the following options to the "Java Options"

 

Recommended Java Options for Java 8:

-XX:MaxMetaspaceSize=1024m

-XX:NewRatio=2

-XX:SurvivorRatio=6

-XX:+UseConcMarkSweepGC

-XX:+CMSParallelRemarkEnabled

-XX:+UseCMSInitiatingOccupancyOnly

-XX:CMSInitiatingOccupancyFraction=75

-XX:+ScavengeBeforeFullGC

-XX:+CMSScavengeBeforeRemark

-XX:+CMSClassUnloadingEnabled

-XX:+UseCodeCacheFlushing

-Dsun.net.inetaddr.ttl=300

-XX:+HeapDumpOnOutOfMemoryError

-XX:HeapDumpPath=<path where a heap dump is written in case of a out of memory error>

 

In cases where the orchestra support has to do a detailed diagnosis of an orchestra system, it is useful to allow the access to the virtual machine by Visual VM (Profiler). This access can be enabled with the following parameters:

 

Debug options for Visual VM

-Dcom.sun.management.jmxremote

-Dcom.sun.management.jmxremote.port=8099

-Dcom.sun.management.jmxremote.ssl=false

-Dcom.sun.management.jmxremote.authenticate=false

 

A detailed description of the configuration of this debugging options (including security options) can be found at Java Management Console
