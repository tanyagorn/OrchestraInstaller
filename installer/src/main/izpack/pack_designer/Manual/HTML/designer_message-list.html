Message lists


Topic content

A Message list is a sequence of Messages.

View of a message list in the Designer

 

A Message list is treated as a single value by the Orchestra runtime.

Each entry in a message list contains it own properties. This feature is used when entries in a message list are searched.

This is especially useful in case of named Message lists which are used in the context of DICOM worklist server and WebDAV Listener. To handle these message lists, within the process model editor a set of scriptslets (predefined Script activities) is available, see DICOM worklist  and Message collection.

Orchestra distinguishes two types of message lists, Anonymous and Named.

Anonymous

This means that the message lists persist as long as the process persists. Afterwards, the message lists as well as the messages are removed by the housekeeping.

Example

Normally it should not be necessary to create Java code to work with message lists. Actually in the process model there is a set of scriptslets (predefined Script activities) to work with message lists. Nevertheless in some areas, e.g. within a Extended Java mapping it might be necessary to explicitely create and modify a message list.

Note: Always use MessageListFactory to create a message list!

MessageStore messageStore = (MessageStore)ServiceRegistry.getServiceInstance( MessageStore.NAME, MessageStore.class );
MessageListFactory messageListFactory = messageStore.getMessageListFactory();
MessageList messageList = messageListFactory.createAnonymousMessageList();

 
messageList.add( MESSAGE );

Named

Named message lists are accessed by a unique name. These message lists persist forever. That means, you have to care about the removing of the contained messages by yourself! The housekeeping ignores these message lists and also the contained messages.

Note: the Scriptlets MessageList.store() and MessageList.get() work with named lists!

Example

QName listIdentifier = new QName( getScenarioIdentifier(), IDENTIFIER );
MessageStore messageStore = (MessageStore)ServiceRegistry.getServiceInstance( MessageStore.NAME, MessageStore.class );
MessageListFactory messageListFactory = messageStore.getMessageListFactory();
MessageList messageList = messageListFactory.getMessageList( false, listIdentifier );
 
if( messageList == null ) {
  messageList = messageListFactory.createMessageList( false, listIdentifier );
}
 
messageList.add( MESSAGE );

Description of the Java object "MessageList"

The message list has only one own property: the time of creation "createDate". The method getProperties() provides the properties that are resided additionally to a message.

 

This figure shows the structure of a message list object together with its messages:

MessageList

|-get(0)  →  Message

|              \-  getProperties()  →  MessageProperties

|                                            |- put()

|                                            \- get()

\-getProperties(0)  →  Properties of the Message list entry, typically search attributes like a name identifying the entry
