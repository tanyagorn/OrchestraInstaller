Installation


Topic content

If you downloaded the Juno zip file follow this instruction to run the software:

Requirements

To run the software a java version 1.8 is necessary on the machine.

Zip file

To install the component download it from orchestra main node. You will receive a zip file, which has to be unziped on a machine in the intended network.

The zip contains four files:

•The orchestra-juno war file

•A properties file , called application.properties

•orchestra-jre-url-extension.jar file

•This documentation as pdf

Orchestra-Juno.war

The war file is the source code of Juno. It contains all necessary libraries and configurations. The file is a so called executable war file, which can be easily started by a console command. You don't need any further frameworks or tools to execute it. It also don't need a database. The war starts the application through spring boot framework. Spring boot makes the application fully executable and adds a integrated tomcat to the war. So if you start the application a tomcat container will automatically be initialized and the Juno application will be started.

Properties file

The properties file is the only way to configure the Juno instance locally. Because you are not able to edit some configurations withing the Juno war file, this must be done here. This means also the logging  and port configurations must be defined in application.properties file.

If you open the properties file, you can see several predefined properties. The file looks as default similar to this:

application.properties example

The first  paragraph contains configurations from spring boot about logging details. The possible framework options to set can be seen on spring boot documentation. An important configuration is the server.port. This is the port on which tomcat container will be started. To change any tomcat properties, look in spring boot documentation. You don't have to add any additional configurations. The application is configured to start instantly.

 

The second paragraph defines the ports of Junos servlet engine. They are set to default, equal to Orchestras ports.

 

The third paragraph contains connection specific configurations. As you can see your specified instance name with

the password and Orchestra configurations.

 

Please don't change the orc. and juno. specific properties. They are individual per downloaded instance. You won't be able to connect to another orchestra then the predefined one.

 

In some cases the IP address must be changed, this depends on your system. The ip adress you can see is the private address of the machine, which hosts Orchestra. In some cases you might need the public IP address of the  network and route it to the correct machine. The property orc.port contains the port, which is waiting for ingoing Juno connections on the Orchestra instance. So it might be necessary to open this port for foreign connection requests.

JRE-URL Extension Jar

This Jar is necessary for a smooth start of Juno. Because Juno is a "little Orchestra" it also needs this jar, which is normally added to Tomcats Java classpath. Because this is not possible in case of Spring Boot initialization, the Jar will be loaded if it's just included in the directory of the war file. The jar will be added automatically to classpath.

Start

To start the software open a shell/cmd in the directory, containing the war and properties files. To start the software a password is necessary. The password was set within the download process from orchestra monitor. Type in the following command:

 

Linux:        orchestra-juno.war --juno.pwd=<your_password>

 

Windows:         java  -jar orchestra-juno.war --juno.pwd=<your_password>

 

 

The software will start automatically and connect to orchestra instantly(if orchestra is available). To see how to access the client of Juno got to GUI entry.

You can overwrite the properties of the application.properties file by typing --<property_name>=<new_value>
