Request appointments by using Query Parameters (Method GET)


Topic content

 

In this chapter we will create a Orchestra REST Provider implementation for requesting  for requesting open appointment-slots by using Query parameters. According to chapter 3.3.5 Request appointments by using Query Parameters (Method GET) we request the appointmentService by using the URL, but this time the appointmentService is implemented by Orchestra and for the Request we use the POSTMAN addon. Do following steps for the implementation.

http://localhost:8080/AppointmentMgr/appointments/doctors/{doctorname}/slots

1.Copy your shell scenario from chapter 5.4.1 Preparation, or enhance your Scenario from the previous chapter.

2.Create a process model.

3.Add the mapping getSlots to your process model.

4.Link all elements in your modell.

5.Open the Properties of your process model. Add  string variables “doctor” and “date” with Usage INPUT.

Figure 93 – Request appointments by using Query Parameters – Process Modell Properties

6.Configure the Input and the Output variable assignment of the Mapping.

 

 

Figure 94 – Request appointments by using Query Parameters – Process Modell

7.Open the REST Service Provider REST_Provider_Appointment.

8.Add a resource to the root resource /AppointmentMgr/appointments.

9.Set to resource path /doctors/{doctorname}/slots.

Figure 95 – Request appointments by using Query Parameters – Resource /doctors/{doctorname}/slots

 

10.Insert a Method GET to the resource /doctors/{doctorname}/slots.

11.Set the Media type of the Response to application/xml, select the Message type openSlotList and set the Result code 200.

Figure 96 – Request appointments by using Query Parameters – Resource /doctors/{doctorname}/slots Response Configuration

12.Add a variable “date” with Style “query” and Type string to the Request.

Figure 97 – Request appointments by using Query Parameters – Resource /doctors/{doctorname}/slots Request Configuration

13.Change technical mapping of the Method GET.

14.Select the process model you created at register Technical mapping.

15.Set the template parameter doctorname to the doctor variable of the process model at register Request mapping and the query parameter date to the date variable.

16.Use application/xml as ContentType, MSG get’s the CURMSG of the process model and HTTPStatusCode is 200 (200 = OK) at register Response Mapping.

Figure 98 – Request appointments by using Query Parameters – Resource /doctors/{doctorname}/slots - Technical mapping of Method GET

 

17.Start your Test engine and select register Service registry at the Testengine-Panel. Copy the endpoint for your REST Provider.

Figure 99 – Request appointments by using Query Parameters – Resource /doctors/{doctorname}/slots – Testengine Service registry.

 

 

 

18.Copy the link for the patients resource to your Postman application set the Method to GET and send the request If every thing is fine you’ll get following response (See screenshot).
