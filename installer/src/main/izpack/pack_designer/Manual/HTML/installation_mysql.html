MySQL


Topic content

The needed SQL Scripts for the Orchestra Installation can be found in

...\Database\MySQL_5.1

 

Important when using MySQL 5.6:

Usually Orchestra uses the JDBC driver for  MySQL Versions, up to 5.5.

When Using MySQL 5.6 you have to  change the JDBC Driver

Therefore download  the correct JDBC driver and replace the MySQL driver in folder /orchestra/WEB_INF/lib

To ensure the Designer finds the new driver you also have to adjust the file orchestra.lib respectively orchestra.bat.Notice: Be aware, on Unix, MySQL case sensitive!                

Because of this, it’s necessary to set the parameter lower_case_table_names to 1. In result, MySQL is no longer case sensitive. A reboot of the MySQL-Server is necessary.

Adaption of MySQL configuration

To configure your database to run Orchestra, you need to change the MySQL configurations. To do so, open the my.ini or my.cnf (e.g. in /etc/mysql or C:\Programme\MySQL\MySQL Server 5.1.) in a text editor and change the settings as shown below.

[client]

default-character-set=latin1

[mysqld]

collation-server = latin1_german1_ci

init-connect='SET NAMES latin1'

character-set-server = latin1

default-character-set=latin1

wait_timeout = 28800

transaction-isolation=READ-COMMITTED

lower_case_table_names = 1

max_allowed_packet = 100M

 

Restart the MySQL Server afterwards. This can be done in the MySQL console, the MySQL System Tray Monitor or directly in Control Panel>Administrative Tools>Services.

 

Parameters can be displayed on the console with the SQL statement:   SHOW VARIABLES like 'lower%';

 

Creation of the Orchestra Database

1.Start the MySQL console as root

 mysql -h localhost -u root –p

2.Create a Orchestra database

 CREATE DATABASE ORC;

3.Create an user for Orchestra

 CREATE USER ORCHESTRA@localhost IDENTIFIED BY  'ORCHESTRA';

4.Grant all rights of the database to the user

 GRANT ALL PRIVILEGES ON ORC.* TO ORCHESTRA@localhost;

5.Log out as root

 \q

6.Log in as Orchestra user

 mysql -h localhost -u ORCHESTRA –p;

7.Switch to Orchestra database

 USE ORC;

8.Change all paths in the mysql__install_all.sql script, which can be found in Database/MySQL_5.1/..., to the path this script is stored. It is appropriate to copy the folder from the Orchestra CD to your system before. Invoke the script afterwards with:

  \. mysql_install_all.sql

 

It can happen that the table ORC_WORK_QUEUE is not set as Engine = InnoDB, this can lead to problems on fast servers.

 

It is appropriate to check if the table has the schema InnoDB:

 SELECT table_name,engine FROM INFORMATION_SCHEMA.TABLES WHERE table_schema=DATABASE();

 

If it has the schema MISAM, it can be changed with this Statement:

 ALTER TABLE ORC_WORK_QUEUE ENGINE = InnoDB;
