Input filter


Topic content

Concept

When dealing with binary data, orchestra has to transfer the binary content (e.g. file) to the internal format of orchestra. In this context two concepts are important. This are De-Serializers and filters. A De-Serializier is a parser, that receives a sequence of bytes and transfers it to the internal message format. Before this parsing step is applied, a set filters can be applied that transform the binary content before it is given to the De-Serializer. Internally this works on basis of so called filter stream. The idea is, that we can nest different filter stream around the origin in order to perform necessary pre-processing of the input data. In this chapter we describe the creation of such a user defined filter.

The generated template works on the idea of a filter stream. The De-Serailizer reads data a byte wise from the underlying filter stream by invoking the read.  Depending on the configuration, within such a filter chain, the user defined filter will also be invoked. This another filter or the De-Serializer itself invokes the read method from the filterstrem. Within the read method a character from the underlying stream is read ( in.read()). In the above example this character is returned immediately, except it is a semicolon. In the latter case, a semicolon is replaced by a '/' sign.
