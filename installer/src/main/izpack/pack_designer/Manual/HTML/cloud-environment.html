Cloud Environment


Topic content

Concept

To enable the best usage of Orchestra in a cloud based environment, the software provides now new functionality. The idea of the new environment is to connect multiple local networks to a global cloud environment, using orchestra. Therefore soffico developed a light weight component, which will be running on local system and communicate with a global main Orchestra installation. To see the architectural concepts go to cloud section in infrastructure chapter. Therefore a component, called Juno is provided.

 

Use Cases

The mechanism can be used in many different cases. It's not forced to run in a cloud environment. To understand the goal of Juno with Orchestra the functionality will be explained in a short use case:

 

•Precondition: An orchestra installation(or several in case of a cluster or cell system) is running in a network. The network may be a cloud environment but can also be your local companies environment.

 

•Problem: Your company has several locations, which have to share information with each other. They are not connected by a VPN connection or something else. For this communication Orchestra should be used. Some channels should connect systems in the locations local network to Orchestra.

 

•Solution:  Orchestra provides a little local application, which can be easily connected to an Orchestra in a foreign network. The component can be installed quickly and is managed from Orchestra's monitor. This little application is called Juno. Juno is not a complete Orchestra, it contains just the channel specific engines. Therefore on the component only channels can be  installed. You can switch every channel you want to a Juno installation, independent of scenario or orchestra node.

Creation

 

For a better understanding a example creation process is explained here:

1.Download a Juno instance in Overview entry of the monitor.

2.Move the downloaded zip file to the local network, which should be connected to Orchestra and unzip the file.

3.Follow the installation guide, which is contained in the zip file or in Juno-Installation topic. If the configurations are correct the component will connect automatically to Orchestra.

4.After a successful connection you can see a new tab in Overview panel, which contains several information about the instance.

5.Now the channel switching can be done in Administration panel. After the selection of a scenario two boxes should be visible. One for Orchestra and one for the Juno instance. Now you can switch channels between the networks by drag and drop.

 

Security

Orchestra and Juno interact in a very secure way. The security mechanisms to ensure the main protection goals

•Confidentiality: The configuration and administration of Juno data can only be done in Orchestra Monitor. This means only an authorized person is able to change processing functionality.

•Authenticity: Juno can only be connected to Orchestra if it is registered in Orchestras Juno registry and if the declared password is correct. Each downloaded instance can only be connected once.

•Integrity: Juno ensures a normal behavior or scenario administration and process execution as it works without a foreign component. The contained data will not be changed during transmission process.

•Availability: Juno ensures a solid connection and data transfer. Also in case of connection problems Juno cares about the correct processing of data. It handles connection loss and reconnects automatically, if a connection is available again.

•Data transfer security: The connection between Orchestra and Juno is a bidirectional secure TLS connection. Juno uses its own protocol to communicate with Orchestra.  

Usage

 

The usage of Juno from monitor is managed in main entry Cloud Overview. This entry has two sub entries:

•The Overview: In this section you are able to disconnect and delete Juno instances or create new instances. Additionally you will have an overview of the actual state of all running instances.

•The Administration: This section realizes the channel switching mechanism. After the deployment of a scenario the channels can be switched to a Juno instance. This administration of the different channels will be done here.
