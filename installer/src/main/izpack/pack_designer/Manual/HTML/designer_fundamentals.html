Fundamentals


Topic content

Working with Orchestra falls into two areas. Modeling and administration / operation. In modeling, interfaces are represented within Orchestra scenarios. Modeling of an Orchestra scenario is done in Orchestra Designer and within in the designer the following core components are found:

•Processes

•Mappings

•Channel

 

When implementing interface scenarios that interact with different subsystems a lot of  problems has to be solved. The first step is to establish a physical connection between Orchestra and the external systems.  This task in general is done by a concept that is called a channel. The next step is a conversion between different data structures. This is very important since different systems normally have different data models, technologies and data structure. In order to cope with this fact a transformation is necessary that translate the data from the source system to the data structures of the target system. For this task, Orchestra provides  transformation engines that can be used to convert data structures. The third step is defining the process flow between the integrated systems. When different systesm have to be integrated, normally the flow of activites has to be defined. Within Orchestra this is done by defining process modells that bring together the building blocks like channels and mappings.
