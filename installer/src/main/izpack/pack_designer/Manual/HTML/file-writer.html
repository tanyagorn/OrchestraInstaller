File Writer


Topic content

Description

This channel is used to write a message in a distinct goal file.

Purpose of object

Writing a message can be done local or remote. Different serializer can be used to create the desired format.

Creation

To create a File Writer follow the steps described in the general description of Channels

Configuration

The dialog to configure the File Writer looks like:

Base view of the channel file writer

Base directory: Choose an appropriate environment entry.

Authentication: (Optional) Choose a user for log in. This user must be created before.

Mode: This channel can:

- Create/Overwrite files

- Extend files

- Create errors if file already exists.

Mapping: The message can be transformed by a mapping. And written in the file per serializer.

Record separator: A string can be inserted here to separate different messages in the file.

How Orchestra works

1.The file name that the output file should have is generated. If a work suffix is used and the check box"Replace file extension" is selected, the current suffix is replaced by the work suffix. If a work suffix is used and the check box is not selected, the work suffix is appended to the current file name. If the file already exists, the work suffix is appended to the existing file (Orchestra does not copy the existing file).

2.If necessary, sub directories are created.

3.The output stream with the same mode as specified in the configuration is created (the file with the file name created above is created).

4.If a mapping is selected, the mapping is now performed.

5.The message is written, if necessary, the separators are added.

6.The outbound stream is closed.

7.If a work suffix is used, it is now removed.
