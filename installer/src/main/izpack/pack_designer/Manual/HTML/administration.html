Administration


Topic content

Concept

The administration dialog provides the possibility to switch the network for channels. Therefore all possible execution environments(either Orchestra or a connected Juno) are available in this view. To switch the channels please select either one scenario or all in the top left corner:

Usage

Administration panel after selecting a scenario. In this case a scenario called DemoScenario

 

As default all available channels are in the column of this orchestra instance. The header of each box is the name of the instance. The type of each channel can be seen by the image, which you also know from orchestra designer. If you don't know any more, which channel should be switched, you can easily open the scenario documentation by button See documentation. Now you can switch the channels by drag and drop them to the intended instance. The channels will be instantly switched to the other network. After you switched the intended channels the view might look like this:

 

 

 

Please pay attention if a scenario is already running! The switched channel will start processing at once.

 

In case of  multiple Juno instances you have the opportunity to filter the displayed instances by opening the filter dialog via Set Filter. In this dialog you are able to hide some instances.

Example filter dialog for Juno instances

 

Use case

To show a example how this mechanism can be used a use case is now explained:

 

Situation: Some information which arrive in main quarter must be spread to some sub locations. Therefore a process scenario will be declared in designer, which contains following channels:

• an ingoing channel, which will be activated by a action in main quarter system.

•an outgoing channel for every sub location, which must be informed.

 

A possible process model of for the described use case is visible in the image below. The example was designed with MLLP channels but that doesn't matter.
