Importing scenario elements


Topic content

When working with  a standard scenario, all elements that have been defined in a scenario library can be re-used. Therefore, the available library elements must be imported into the current scenario. The required function can be found under the menu "File/Include from library"

Importing an element

 

After this function is selected to following dialog appears:

Select a scenario library

In this step, a scenario library is selected. From this library, in the following elements that have to be imported can be selected. The according dialog can be seen in the following figure:

Choose scenario elements for import

 

Whenever you select an element to import (e.g. MessageMapping) the Orchestra Designer checks if there are depending scenario elements that have to be imported as well. In the above example the Mapping references the message type mt_sample. Therefore when the mapping is imported ( State Import) the message type must be imported (State Required). When all elements have been select, the import proceeds with clicking the next butte. When clicking to next, the import starts and Orchestra displays a progress page that shows what currently is going on.

Progress page during import

After that step, all elements have been imported and are now available in the current scenario. In the scenario tree, the imported elements are marked with the suffix "(Imported)"

This looks like follows:

Imported scenario elements

 

 

Note: The imported elements can be used in the designer as all other elements. The imported elements are not protected against changes. This may lead to unintended effects. All changes to a scenario element will be overwritten during the next update from the underlying scenario
