Orchestra SSL configuration

Orchestra provides all ingoing HTTP connections (Orchestra has the server role) over a library to jetty. Connections with HTTPS use a central public/private-key pair. This is defined in the basic configuration of Orchestra.

The following basic parameters are preset:

•location:                configuration directory of Orchestra (WEB_INF\classes\config)

•deposition type:        Java keystore, all common keystore formats like JKS or PKCS12 are supported

•keystore password:        The passwords for the keystore access are deposited in the Orchestra configuration. All passwords are saved encrypted. The encryption is done with the method                                "PBEWithMD5AndDES"

The configuration for the usage of the server certificate are made in the file orchestra_configuration.xml. The following parameters within the group SSLSettings can be set.

Parameter

Description

KEYSTORE

Location of the keystore, this has to be inside the classpath of Orchestra, the default value is config/orchestra_keystore

KEYSTORE.PASSWORD

Password for accessing the keystore, encrypted according to "Orchestra password encryption"

KEYSTORE.KEYPASSOWRD

Password for accessing the private key, encrypted according to "Orchestra password encryption"

KEYSTORE.TYPE

Type of the used keystore, possible values are „JKS“ (Java keystore) and „PKCS12“ (Keystore in PKCS12 format)

Exchanging a certificate

Using the command line

create a private key

openssl req -new -x509 -extensions v3_ca -keyout cakey.pem -out cacert.pem -days 3650

The password is "ORCHESTRA"

composing via openssl

let key and certificate be composed at invocation

openssl pkcs12 -export -inkey cakey.pem -in cacert.pem -out cakeycert.pkcs12 -name orchestra -noiter -nomaciter

convert pkcs12 container into a Java keystore

keytool -importkeystore -srckeystore cakeycert.pkcs12 -srcstoretype pkcs12 -srcalias orchestra -destkeystore orchestra.keystore_soffico -deststoretype jks -deststorepass ORCHESTRA -destalias orchestra

Configure the keystore in Orchestra

The keystore has to be in the config directory of Orchestra! It is not possible to use an absolute path outside Orchestra!

 

This is the configuration that has to be set in orchestra_configuration.xml

<?xml version='1.0' encoding='ISO-8859-1'?>
<configuration>
  <service name = "MasterdataService"  
            class = "emds.epi.impl.baseservice.OrchestraBaseService" >
    <parameters>
            <group name = "SSLSettings">
                <parameter   name = "KEYSTORE"               value = "config/orchestra.keystore" />
                <parameter   name = "KEYSTORE.PASSWORD"         value = "0008464e5a525a4c4e550006726e716264630000001065d6edb79621528455690d4dd418c355" />
                <parameter   name = "KEYSTORE.KEYPASSOWRD"       value = "0008464e5a525a4c4e550006726e716264630000001065d6edb79621528455690d4dd418c355" />
            </group>        

 

The encrypted values for PASSWORD/KEYPASSOWRD can be generated with the scripts gen_passwd from Orchestra (look here for more information: How to use gen_passwd script)

Using portecle

You have to download portecle first!

 

1. Copy <path to tomcat>\webapps\orchestra\WEB-INF\classes\config\security\orchestra.keystore

2. Execute cmd as admin: execute portecle.jar

3. Open copied Keystore in Portecle (Password: ORCHESTRA)

4. Delete old key pair from keystore (alias orchestra)

5. Import key pair

6. Select key pair file

7. Enter password for private key

8. Select key pair for import

9. Enter alias for key pair: orchestra

10. Enter new password for key pair: ORCHESTRA

11. Save keystore

As we have used the same passwords than deposited in the orchestra configuration, no changes have to be made here.
