Process


Topic content

The integration of heterogeneous system often require a complex interaction between different systems. Since there is no standard or typical pattern that can be used Orchestra used a flexible approach to based on the standard notation BPMN. BPMN is a graphical language that can be used to define a flow of activities and tasks. A first example of such a process model is given in the following picture.

process modell

 

When the external System represent by the channel "CH_Ticket_DBInb" sends a message to Orchestra, a process instance is started that executes that steps defined by the process model. First of all, a mapping is executed that translates the incoming message to the required target format. After that, the data is send to an external system by SOAP-Protocol (CH_SOAP_Outbound). As a last step,  an external database system is updated.

 

A closer look to this process model reveals that error handling can be also modeled within a process.  Whenever an error occurs, an exception flow is started that creates an error log entry in an underlying database and log file. The exception flow in this example is defined by the lightning symbol attached to every of the the main activities.

 

The approach of used a model based approach like BPMN is very flexible since a lot of different scenarios can be implemented in this way. There are no limitations and restrictions and a lot of complex problems can be solved using this modeling approach.
