Lookup Tables


Topic content

To use Lookup Tables in the Orchestra designer for testing, the runtime must be started!

Concept

The Lookup Table (short LUT) is used to reduce processing time for Orchestra. Basically, the LUT contains data or results from complex calculations needed by a mapping, which was done beforehand. By keeping the results in the LUT, when Orchestra needs the values, instead of having to do the calculations, it can just refer to the LUT and retrieve the values from it, bypassing the calculations. In Orchestra a LUT is designed as a kind of conversion table, structured as key-value pairs.

 

Lookup Tables can be used directly in an Orchestra graphical mapping. To use a Lookup Table in an Extended Java Mapping it is necessary to create a proxy in a Java Source.

Creation

To create a Lookup Table select the group Lookup Table in the tree of scenario elements, click the right mouse button and select the item Create lookup table.

Then the dialog Create lookup table pops up.

1.First you enter an appropriate name for the new look-up table and optionally a short description.

Dialog to name and describe a new look-up table

2.After clicking the Finish button the Lookup Table is created and the configuration panel is opened.

Configuration

Orchestra supports four data sources for LUT usage.

Clicking the drop-down menu button and choose one of the following data source

•Internal Table - simple static key-value table stored inside a scenario. It can not be changed at runtime.

•Internal Table with regular expressions -extends the simple table by regular Expressions

•External Property file - pre fetched file cached on scenario activation. A reload on file change can be configured.

•database table - using a database to manage and maintain the LUT

The configuration catalog's of these data sources differs and is described below in detail.

If you choose Internal Table or Internal Table with regular expression, you get access to the internal look-up table where you can add source and target values by typing values into the columns as illustrated in the following figure.

If an External Property file is your choice you need to set a file name, a base directory and a reload strategy as shown in the following figure.

To define a base directory you need to assign an environment entry of type filepath.  ( Please see Environment Entry - Filepath for detailed information).

To set up a data source database table you need to assign an Environment Entry of type database connection and to specify the table and the key and value columns.

Finally, you can define a standard behavior what should happen if the requested value is not contained for each kind of Lookup Table.
You can choose between throwing an error or use a default value . Set the default value by typing into field Default value and press enter or save your changes.
