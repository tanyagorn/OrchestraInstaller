Service Client


Topic content

Concept

Category types

Purpose

Service declaration, as one of crucial concepts of Orchestra, is Service Declaration used either to expose interface to external world (server mode) or to use interfaces available outside of Orchestra (client mode). Note that when service declaration is used in server mode, you also have to implement service declaration in Service Provider. However, we are using service declaration in most cases when we want to create service client.

Creating

Manual creating

In order to create service declaration please navigate to scenario elements -> Service Declaration tree, right click and select Create:

Creating service declaration from context menu

Creation dialog will be shown where you have to set Name and, optionally, Description values:

Creating new service declaration

Click on Next and as a final step you will be asked to select Category:

Selecting service category

For details about category types please refer to Category types.

Importing WSDL

Alternative and more convenient way to create service declaration (compared to manual creating) is to import WSDL file. Importing WSDL will not only create service declaration, but will also create message types and XML Schema based on WSDL. Additionally, operations and bindings that are defined within WSDL will be created with settings you choose.

To import WSDL please navigate to scenario elements->Service Declaration, right click and select Import WSDL:

Importing WSDL from context menu

Simply navigate to WSDL file and click Next:

Selecting WSDL file

Note that WSDL import can be done both from file and from URL. Sample Calculator.wsdl can be found in resources from Downloads & Examples.

Next dialog will ask for specifying category, service and port type to be used:

Import WSDL settings

For details about category types please refer to Category types.

Naming allows you to select:

- Use short names for message types

- Add namespaces to message type names

Click on Next button to start importing. Import status should be shown:

Import WSDL status

Click on Finish button. You should then have service declaration structure visible in the tree:

Service declaration tree

Creation of message types, XML Schema and environment entries is done automatically. If you double click on service declaration, that is created from importing WSDL, following panel is shown:

Service declaration panel

All operations defined in WSDL are created and also bindings that use the operations. Details can be found in Operations and Bindings sections.
