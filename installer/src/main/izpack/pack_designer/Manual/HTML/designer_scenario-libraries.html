Scenario Libraries


Topic content

One fundamental principle in informatics is reuse. This is a very powerful principle that helps to reduce the development effort. Within orchestra, scenario libraries are the basic mechanism that allows a reuse of commonly defined scenario elements in different environments. A scenario library is a scenario that only supports a subset of elements. The elements of a library scenario can be imported into other scenario and therefor allow a reuse of elements.

 

Currently the following elements can be maintained in a library scenario

•XML Schema

•Message types

•Message mapping

•Lookup table

•Environment entries

•External archives

•Signals

•Java sources

 

Channels and security settings are currently not supported.

 

Note: Scenario libraries work on basis of a copy mechanism. Every element that is imported is a complete copy of its original. This fact is very important, since a scenario must be executable in the ochestra runtime independent of the existence of the scenario library.
