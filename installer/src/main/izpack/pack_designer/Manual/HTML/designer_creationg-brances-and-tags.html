Working with branches and tags


Topic content

When working with a version control system, it is necessary to mark a version of a scenario as  "release" version. This is necessary since on basis of such a mechanism different source lines of the scenarios can be maintained in parallel. The current development in the scenario is done in the trunk version. When a released version has to be changed (bugfix), it is not useful to change the development branch. In this case it is wise to do only little changes on the original released scenario. In order to do exactly that, you have to mark ensure that you exactly now the state of the scenario at "release" time. This is done with branches and tags. When we talk about trunk, branches and tags, the following definitions are useful.

 

•Trunk would be the main body of development, originating from the start of the project until the present.

•Branch will be a copy of code derived from a certain point in the trunk that is used for applying major changes to the code while preserving the integrity of the code in the trunk. If the major changes work according to plan, they are usually merged back into the trunk.

•Tag will be a point in time on the trunk or a branch that you wish to preserve. The two main reasons for preservation would be that either this is a major release of the software, whether alpha, beta, RC or RTM, or this is the most stable point of the software before major revisions on the trunk were applied.

 

Creating a tag/branch

When a tag or branch should be created, the history dialog has to be displayed. This can be done by invoking the context menu on the scenario element tree. This is shown in the following figure.

Open the history view

 

The history dialog displays the evolution of a scenario over the time. The corresponding view is shown in the following diagram.

The history view

 

The history panel offers  a context menu that can be used to create tags and branches. The context menu is shown in the following diagram.

 

•Create tag: Create a new tag in the version control system. The selected entry that defines the basis for the tag is the selected line in the table.

•Create branch: Create a new branch in the version control system. The selected entry that defines the basis for the tag is the selected line in the table.

•Delete tag: Delete a tag from the versioning control system

•Export scenario: Export a copy of the scenario exactly from the point in time denoted by the entry of the history panel

 

Switching to a tag or branch

If you want to switch to a previously created tag or branch, please use the context menu in the scenario element tree. The menu is shown in the following diagram.

Swith to a branch or tag

 

When this function is selected, the following dialog is shown.

Select a version of the scenario

 

Within this dialog a version of the scenario can be chosen. When the operation is confirmed ("ok") the local copy of the scenario is set  to the tag/branch/trunk that has been selected.
