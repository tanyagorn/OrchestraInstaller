Message lists and the method searchAll( SearchPredicate searchExpression )

Question: How do I create this search expression?

A XPath expression would be obvious, but I don't know how to create a variable that implements the SearchPredicate interface and contains a XPath expression at the same time.

Answer: The method searchAll() does not search inside the messages of a message list, but only on the properties of the entries from the message list.

Be aware of the following:

1.A message list has more properties available

2.Distinguishing volatile and persistent message lists

1. A message list has more properties available

An entry of a message list contains two objects:

•the message

•a properties object

But: The message itself has also properties! (time stamp when the message was created, file name, Content-Type)

 

Structure of a message list with three entries:

--------[1]--------------------------[2]--------------------------[3]-----

            /     \                                        /     \                                        /     \

   MSG1   PROP1                    MSG2   PROP2                    MSG3   PROP3

        |                                              |                                              |

  PROP_MSG1                        PROP_MSG2                       PROP_MSG3

 

The searchAll method searches the properties-objects PROPx of the message list, but not the properties from the contained messages (PROP_MSGx)!

2. Distinguishing volatile and persistent message lists

•volatile message lists: the properties in the memory are searched and the matching entries provided as a result

•permanent message lists: the database table orc_message_list_entry_prop is searched for matching entries

Example for the usage of searchAll

String referenceDate = orcjava.briefe.getCurrentDateMinus3AsString();
List<MessageListEntry> result = messageList.searchAll( new ComparePredicate( "DATE", "<", referenceDate ) );

List of possible search predicates:

All predicates are in namespaces emds.epi.decl.server.message.

ComparePredicate (String propertyName, String operator, String value);

a compare condition: <propertyName> <operator> <value>;

operator can be one of =, <=, >=, >

PropEmpty(String propertyName)

Checks if the given property is empty or null

PropertyPredicate(String propertyName, String value)

Checks if the the value of the given property is equal to the given value

RangePredicate(String lowerBound, String propertyName, String upperBound)

Checks if the the value of the given property lies between the to limiting values. If the value of the property equals one of the bounds the condition is true.

StartsWithPredicate(String propertyName, String initialString)

Checks if the the value of the given property starts with the given initial string value.

CompoundPredicate( SearchPredicate lhs, String op, SearchPredicate rhs )

Two predicates combined with and; so both predicates must evaluate to true to fulfill the CompoundPredicate. Therefore the parameter op must be "and".
