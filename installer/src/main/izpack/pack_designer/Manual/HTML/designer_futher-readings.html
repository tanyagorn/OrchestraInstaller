In this section we provide some useful tip's and tricks when working with scenario libraries.

 

•An imported scenario element is always stored as a copy of the original

•The scenario always works on the local copy. Even if the scenario library was modified, this modifications are not automatically re-imported. The importing scenario works on its own settings until an update is done

•When a scenario library is deployed to the runtime, only changes of the environment settings can be automatically pushed to the depending scenarios

•An update of a mapping, lookup table ( anything but an environment entry) can only be done within the designer. This is a well designed feature, that ensures and protects against failures.

•When a scenario is deployed to the runtime and the scenario library is not available the local settings of the environment entries are used.

•A scenario library must not be activated. Since a scenario library has no active elements (like channels) the activation is not relevant for a scenario library

•When working with java sources in a scenario library, please ensure that all designers use the same version of the JDK. This is important, since the a scenario contains the compiled byte code of the java sources. The may occur errors when different java versions are used.

•If you want to change the type of a scenario from standard to scenario library you have to change this settings manually in the file system. The following steps have to be done

oCopy the scenario into the folder of the scenario libraries

oOpen the file "props" and set the property scenarioType to the value of 1
