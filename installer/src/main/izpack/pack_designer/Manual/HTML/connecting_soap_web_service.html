Connecting to an external SOAP web service


Topic content

Description

Orchestra application provides users with a  clean and "easy to use" functionality for integrating external web services. Using this functionality user can easily combine data from multiple Web services  and incorporate them to their process models, even if the systems themselves are incompatible. Providing an ability to import WSDL  XML-based format for describing web services, Orchestra allows user to  easily configure and integrate  with external web services within just few steps.  

Problem to Solve

Imagine yours business process is depending on the information about Weather-Forecast  for some specific region. Since there are several publicly available web services that can serve you with this kind of information, all you need to do is to configure your Orchestra application to be able to effectively exchange data with the service and incorporate this data to your process model. The following example will describe how to periodically pull the forecast data for selected city and store this information to the xml file.

Step Overview

In order to successfully connect to external SOAP Web Service service, several steps need to be followed.

1.Choose the Web service and download service WSDL file

2.Create Orchestra Service Declaration by importing previously downloaded WSDL file

3.Optionally, apply additional configuration setting for created Service Declaration

4.Incorporate created service declaration to your process model

5.Start process model in order to successfully exchange data with the service

Prerequisites

Connecting to an external SOAP web service using Orchestra app does not require any previous knowledge of web service implementation. The extended knowledge about WSDL format is not required but it could be useful in case you need to apply additional configuration for web service you are using.

Used Concept

Orchestra introduces Service Declaration concept which can be used in order to connect to an external SOAP web service.

Steps

The following text will give you a detail step-by-step explanation how to connect to external Weather-Forecast service and use data retrieved from it.

 

Step 1 - Choose Web service and download service WSDL file

Choose one of the publicly available web services, for example Weather-Forecast. Simply download a WSDL file from publicly available internet address. In our example we will use following WSDL file

http://www.webservicex.net/globalweather.asmx?WSDL.

 

Step 2 - Create Orchestra Service Declaration by importing previously downloaded WSDL file

•Right Click on the Service Declaration option from Orchestra designer main menu

•Select the option Import WSDL

Service Declaration - Import WSDL option

•After clicking on the Import WSDL option you will be presented with the Import web service description wizard dialog

•Select the WSDL file and click on the Next button

Note: in case you are using WSDL files with nested complex types you can simply type HTTP address of WSDL file in WSDL input field.

•New Screen will be presented to you to adjust additional settings. Click on the Next button to continue.

•After successful importing of the WSDL file you will be presented with the status screen. Click Finish.

Service Declaration - import WSDL dialog

•Now the Service Declaration is created and configured based on WSDL file definition.

 

Step 3 (Optional) - Apply additional configuration setting for created Service Declaration

After successful import of the WSDL file, several different Orchestra entities have been created based on a WSDL web service definition.

You will find new Service Declaration element together with corresponding message types and XML Schema elements. Beside Service Declaration element the corresponding environment entries have also been created. See the image below.

Service Declaration - Generated scenario elements

As you can see from the image above, Orchestra will automatically create all scenario elements required for successful connection with external web service. IF there is a need you can manually change created elements. Following steps will explain how to modify data produced by importing WSDL file.

•Double click on the Service declaration element ()to open the configuration panel.

•On the left side of the opened configuration view you will find Bindings and Operations configurations.

•Under operations element you will be able to see list of web service available operations. See image below.

Service Declaration - Operations Configurations

•Double click on the Ports element to apply additional changes for your Service declaration.

•You will be presented with the configuration view containing following configuration settings. See image below.

Service Declaration - Ports Configurations

•On this page you can modify various configuration settings for your web service, Port, URL, Connection Timeout etc.

For more details about web service security configuration refer to Web Service Security in SOAP web service calls best practice story.

•After you have successfully configured your Service Declaration of external SOAP web service, start using web service inside your process models.

 

Step 4 - Incorporate created service declaration to your process model

For purpose of testing connection to a external forecast web service we will use following example.

Using the previous created Service Declaration we will fetch the data about weather forecast for specific place name and store it to a xml file.. In order to do this we will incorporate Service Declaration inside existing process model.

•Select your process model from Orchestra main menu.

In order to create new process model go to Process Model reference manual

•Click on the Service Declaration element and place it to your process model diagram.

Scenario Declaration - process model example

Note: For this example we will use a simple process model implementation. The model contains File Listener as an inbound channel used as a starting event and File Writer as an outbound channel for storing weather forecast result. File Listener channel is configured to periodically listen for file changes inside defined file path on the local system. The file contains name of the  city and the country and  will be used as an input parameter to our Service Declaration. The file can be created based on the message type inside our service declaration which is generate by importing WSDL file.

Service Declaration - Create xml input file

•Connect the Service Declaration to the service call element

•Configure process model and select appropriate method for fetching weather forecast for selected place name.

Service Declaration - service call configuration

•Select the output xml file for storing retrieved data from Weather forecast service (File Writer channel configuration).

 

Step 5 - Start process model in order to successfully exchange data with the service

Using the process model created in previous step start collecting the data.

•Start test environment to collect the data.

•Place file with name of the city and the country which can be picked up by the File Listener channel.

•Open the file created by File Reader Channel. File should contain forecast data for selected city.

•Check xml file and verify the data.

 

Note : The process model used for this example is just one of the possible usage of Service declaration concept. You can utilize this concept in so many different ways with various external web services based upon your process models.

Summary

The previous example shows how to easily connect to external web service by using concept of Service Declaration  and effectively exchange data within your process model. Using the import functionality gives you the ability to integrate Orchestra application with different web services in just few steps. Combined with other Orchestra features you can now utilize various functionalities provided by different external web services.
