Byte translation filter


Topic content

Description

This filter is available as input as well as output filter.

The filter translates the bytes to a sequence of bytes according to a given translation table.

Creation

See the description in Stream filters

Configuration

To configure the filter double click on its name. A dialog appears where you can configure the filter:

translation filter to replace a single carriage return by carriage return and line feed

•Name In this field the user may change the name of the filter step to a descriptive name

•Description Here the user may enter an additional description

•Table The example shows how to define a filter which replaces a single carriage return (Hex code 0D) with the two bytes carriage return (hex code 0D) and line feed (hex code 0A). The filter allows to add an arbitrary amount of translations. It might even be useful to remove some bytes by replacing it by an empty string. In the given replacement string the non printable characters tabulator (hex code 09), carriage return (hex code 0D) and line feed (hex code 0A)  are given using \t\r and \n.

•Remove Button To remove a byte translation entry select it and click the Remove button.

•Add Button To add an new byte translation entry click the button Add. Then a dialog to enter the new values appears:

 

Example

enter a new byte translation converting a sharp s to double s

 

This example shows how to replace the special german character sharp s (Hex code df in Latin1 and Unicode) by a double s. Note that it is possible to enter not printable source bytes  through its hex code but you can not enter not printable replacements apart from \t\r and \n!
