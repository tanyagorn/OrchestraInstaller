Serializer


Topic content

Concept

All Channels writing a byte stream need a serializer to create the content from a message.

Note that the terms Inbound channel and Outbound channel don't refer to the direction in which the data flow but to to initiator of the call. So an inbound channel might write data (e.g. the HTTP GET Receiver) and therefore needs a serializer.

E.g. if you want to write a CSV file using the File Writer, you need a  to transform the DOM tree of the structured message into the byte stream written to the file.

Some serializers need information from special Message types, one example is the Type based serializer. When configuring these serializers the user always has to select an appropriate message type.

There are serializers which need no message type to work, e.g. the the XML writer. And finally some serializers may use a message type but can work also if no message type is selected, e.g. the CSV writer.

Creation

A serializer is always created when configuring a Stream channel.

Basically the user has to select a serializer, select an appropriate message type (if necessary) and then click on the button marked with ... to configure the serializer (if necessary).

If no serializer is selected so far or only the binary writer is selected and the user selects a message type associated with a particular serializer, then that serializer is automatically selected too.

Below the line for selecting the serializer you will find a line where you can select a Stream filter by double clicking the arrow -->

output filters

This way you also may build a sequence of filters.

To configure a filter the user has to double click on the name in the sequence.
