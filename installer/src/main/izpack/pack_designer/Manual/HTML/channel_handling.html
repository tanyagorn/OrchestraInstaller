Channel


Topic content

The status of each channel of a scenario can be monitored separately. You can start, stop and reset Channels independently (e.g. a file channel that should not be used in the runtime environment).

A right-click on the desired channel in the table opens a context menu with some control options.

Channel Summary of a Scenario

A channel can have the following states:

•stopped                

•started                

•undefined        

Adapter can be stopped, started or their state can be reset here.

The last column is used for cloud version. It names the instance on which the channel is running. For further information see cloud environment chapter.Additionally, you can get detailed information about a channel by clicking on it (for example URL, number of ingoing/outgoing messages).
