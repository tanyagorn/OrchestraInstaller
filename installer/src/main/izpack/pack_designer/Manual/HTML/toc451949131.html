Update an existing Patient (Method PUT)


Topic content

In chapter 4.5.3 Listing all patients in XML (Method GET) we requested a list of all patients in XML. Now we want to update an existing Patient, so we use PUT as Methode for the URL

http://localhost:8080/AppointmentMgr/appointments/patients

 

For this Request we have to tell the the server, for which patient we want to update, so we have to set the Bodycontent

<patient>

<id>1</id>

<name>NewPatientName</name>

</patient>

of our Request. We also have to inform the server which Content-Type we send in our Bodycontent, so we have to add the header Content-Type: application/xml

1.Copy your shell scenario from chapter 4.5.1 Preparation.

2.Create a Message Type patient for the Bodycontent and add the child nodes.

Figure 12 – Update an existing Patient – Message Type

3.Create a TragetOnly mapping for the Message Type patient with input string variables “name” and “id”.

Figure 13 – Update an existing Patient – TargetOnly mapping

4.Open your process model.

5.Add the Rest Service Client, the TargetOnly mapping and a call activity to your process model.

6.Link all elements in your modell.

Figure 14 – Update an existing Patient – Process Modell

7.Now you can configure the Input/Output variable assignment of the TargetOnly Mapping.

Figure 15 – Update an existing Patient – Input/Output variable assignment TargetOnly Mapping

8.Now you can configure your call activity for the AppointmentMgr-Service. Open the register Service method and select the PUT-Method for /appointments/patients.

9.Assign the Response MSG of the call to the CURMSG of the process modell.

10.Assign the Header-Parameters ContentType “application/xml” and the Bodycontent MSG (Use the MSG you created in the TargetOnly mapping).
