Implementing REST clients in Orchestra


Topic content

•REST Service Provider is used to implement a REST Service within Orchestra(since release 4.5.5.0)

•REST Service client is used to implement a client to an external REST Webservice(since release 4.5.6.0)

•SecURLty configuration is similar to the model that is used in HTTP channels(HTTPS, Credentials)

•Predefined WADL service description can be imported as „REST service declaration“ in order to define the service interface of a client or a server

•Data can be accepted/returned in different representations (JSON, XML, etc.)

•The Orchestra documentation provides two best practice documents that describe indetail how to implement REST client and REST service with Orchestra
