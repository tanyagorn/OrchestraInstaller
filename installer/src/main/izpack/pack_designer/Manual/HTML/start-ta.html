Start TA


Topic content

Description

Ensures that a transaction has not been executed with the given ID within this scenario. The given transaction ID is stored persistent by the Orchestra runtime.

Depending on the configuration of this Activity an error can be generated if the transaction has been performed once.

Create

1. Create a process model

2.Then point the mouse on "process", as default, this window is on the right side of the Orchestra window.

3.Click on the folder Business-Transaction to see its content.

4.Click on "start TA".

5.Click on any place in your process model.

6.Integrate it in process schedule, by clicking on the start TA icon and drag and drop the arrow to the to be connected item.

7.Connect it  by clicking on the start TA icon and drag and drop the arrow to the intended service. A dashed arrow should appear.

 

Icon

 

Configuration

Transaction handler

There are two mode defines the behavior of the transaction handler.

1. Generate runtime error if transaction already exists.

2. Return the state of the transaction as a result of the activity.
