Installation

Copy the Orchestra folder from the zip-file into TOMCAT\webapps. If you rename the Orchestra folder, the Path to the URL changes. On standard values, Orchestra is later accessible on http://localhost:8080/Orchestra .

Configuration

Open TOMCAT\webapps\Orchestra\WEB-INF\classes\logging.properties with a text editor and define line 40 as displayed below.

# Default global logging level.

# This specifies which kinds of events are logged across

# all loggers.  For any given facility this global level

# can be overriden by a facility specific level

# Note that the ConsoleHandler also has a separate level

# setting to limit messages printed to the console.

# There are seven Levels: SEVERE (highest value), WARNING, INFO,

#  CONFIG, FINE, FINER and FINEST (lowest value)

.level = WARNING

 

############################################################

# Handler specific properties.

# Describes specific configuration info for Handlers.

############################################################

 

# default file output is in user's home directory.

java.util.logging.FileHandler.pattern = C:/Orchestra/logs/Orchestra_%g.mlog

java.util.logging.FileHandler.limit = 250000

java.util.logging.FileHandler.count = 4

java.util.logging.FileHandler.formatter = emds.log.context.StandardFormatter

 

It is necessary that the directory exists and the Tomcat User can write on it. Use slashes instead of backslashes when describing the path. 'Orchestra_%g.mlog' is the name of the log-files. The parameter %g generates the numeration on the log-files.

FileHandler.limit describes the maximum size of the log-file in bytes.

FileHandler.count describes the maximum amount of log-files.

Line 32 describes the accuracy of logging. We recommend to keep this value on WARNING.

Other values are:

SERVE

Indicates only serious failures.

WARNING

Indicates also potential problems

INFO

Indicates also  informational messages

CONFIG

Indicates also static configuration messages

FINE, FINER, FINEST

Indicates detailed messages till every single message

 

Open TOMCAT\webapps\Orchestra\WEB-INF\classes\config\environment_setting.xml with a text editor and choose your database by setting 'enabled' on true. Change the parameters of the runtime.url(network-address of the database) and the database-user if necessary.

The parameters runtime.user and runtime.pwd are encrypted values for the database. Predefined is ORCHESTRA ORCHESTRA. If you have chosen other log-in-values, generate the encrypted values with gen_password.cmd.

<group name = "database" enabled="true">

         <parameter name="runtime.db.typ"          value="MYSQL" choice="HSQL|ORACLE|MSSQL|DB2|MYSQL|Derby|INFORMIX|PostgreSQL" />

         <parameter name="runtime.url"             value="jdbc:mysql://localhost:3306/Orchestra" />

         <parameter name="runtime.user"            value ="00084456584244414444000670766f6c6872000000106c0eed592c3583eaf6515b3ce5df04ca" />

         <parameter name="runtime.pwd"             value ="0008414e43434d54495400066d6e746d716b00000010c70877ffae9311c08ffbbadec4e23bd1" />

   </group>

 

For generating such passwords the tool "gen_passwd.cmd" can be used. It is located in the directory "startscripts\gen_passwd.cmd".

(Before this tool can be used, a configuration of the environment is necessary. Please referer to Configuration of the designer environment)

 

Start gen_passwd from the command line as follows:

gen_passwd.cmd <YOUR_PASSWORD>

How to use gen_passwd script

 

Copy the encrypted values to the environment_settings.xml.

 

Starting Orchestra

Orchestra is automatically started with Tomcat. Orchestra is accessible at http://localhost:8080/Orchestra. Log in with admin admin.
