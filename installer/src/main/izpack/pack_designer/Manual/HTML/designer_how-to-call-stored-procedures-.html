Stored-Procedures Aufruf in Orchestra

Dank  Stored-Procedures  kann  Anwendungslogik direkt in einer Datenbank implementiert werden. Stored-Procedures  können in  Orchestra  über  Datenbank-Outbound-Channels  aufgerufen  werden. Jedoch ist der Umfang auf Prozeduren ohne Rückgabewert begrenzt. Dies schränkt die Verwendung

ein, wenn bspw.  der Rückgabewert einer Stored-Procedure innerhalb  eines Mappings verwendet werden soll.

 

Alle drei Beispiele sind in Form eines Java-Channels realisiert und zeigen die wesentlichen Schritte für

den Aufruf der Datenbankprozeduren. Hierbei wird auf Funktionen zurückgegriffen, die in der Orchestra-Customer-API implementiert sind.

 

Die verschieden Prozeduren beziehen sich auf folgende Datenbanktabelle:

 

CREATE TABLE [dbo].[sample_proc](
 [id] [varchar](50) NOT NULL,
 [col1] [varchar](50) NULL,
 [col2] [varchar](50) NULL,
CONSTRAINT [PK_sample_proc] PRIMARY KEY CLUSTERED ([id] ASC))
