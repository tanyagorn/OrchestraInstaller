Installation of the worklist handler


Topic content

The worklist handler is designed as a web application that is embedded within the orchestra runtime. All administrative settings are done over the orchestra monitor. The normal work of a "user" is done in a seperate worklist application that can be reached over web.  

 

The following URL's can be used to connect to the worklist ui:

https://<server>:8090/humaninteraction

https://<server>:8091/humaninteraction

Configuration of the worklist-handler-ui

The configuration of the worklist handler can be found under "Settings/Environment settings/Worklist integration"

Group "worklist integration"

enabled

If set to true, the worklist componente is active. Otherwise it is inactive.

path.ui

Path to the folder that contains the web-app of the worklist ui. Per default this path is automatically adjusted to a directory relative to the orchestra installation

 

completion.max_retries

Maximum number of retries that will be executed when the worklist handler ui tries to send a completion event back to orchestra

completion.retry_delay

When a notification to orchestra fails, the next retry time is computed by added this value to the current system time

completion.retry_timer_wait

This time defines the sleep interval between two checks of items that have to be retried

escalation.timer_wait

time that the timer that handles the escalation waits before it checks for escalated items

 

email.enabled

If set to true, orchestra sends an email to the recipient of a worklist item. The email address of the recipient is given in the description field of a credential

email.from

sender email address

email.skipLogin

If set to true, the login dialog will be skipped when the worklist ui is opened by an external link from a mail.

email.smtphost

Name of the SMTP server that is used to send mails

email.smtpport

Port of the SMTP server

email.smtpuser

Technical mail user that is used to connect to the SMTP server

email.smtppwd

Password of the SMTP user

email.usessl

true: use a SSL connection, false: use a standard TCP connection

server.entry.url

An URL that is embedded in the mail sent by orchestra. This URL is used as link back to the worklist ui. Sample: http://host:port/humaninteraction

email.subject

The subject of the generated email. The subject can contain parameters that are substituted before the email is sent. Please see the following section for a list of available parameters.

email.body

The body of the generated email. The body can contain parameters that are substituted before the email is sent. Please see the following section for a list of available parameters.

token.expiration

This timeout defines how long an external form is allowed to send data to the worklist server. If it needs more time an error is raised.

ui.title

The headline that is displayed when working with the worklist ui. Use the parameter $(CURRENTUSER) when the user name should be displayed in this headline.

 

 

Note: Use "\n" to denote a line break.

 

Parameters that can be used in email.subject and email.body:

 $(WORKITEM.NAME)        -        The name of the worklist itm, given as parameter "ItemName" in the workflow activity

 $(WORKITEM.ID)        -        Unique identifier of the worklist item

 $(WORKITEM.DESC)        -        A descriptive text given as parameter "ItemSubject" in the workflow activity

 $(WORKITEM.DATE)        -        Creation date of the worklist item

 $(WORKITEM.PRIO)        -        The priority of the worklist item given as parameter "Priority" in the workflow activity

 $(WORKITEM.URL)        -        The URL that is used as entry point for this worklist item into the worklist ui
