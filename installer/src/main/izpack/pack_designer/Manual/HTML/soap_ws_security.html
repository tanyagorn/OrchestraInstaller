Using Web Service Security in SOAP web service calls


Topic content

Description

Nowadays, SOAP web services use different security method to ensure confidentiality of exchange data.  By using the WS-Security standard, Orchestra provides the ability to leverage emerging XML security technologies to address web services security requirements . Orchestra Designer provides you with a simple UI to configure web service security parameters of your  SOAP web service calls.

Problem to Solve

In the Connecting to an external SOAP web service best practice story we explained how to successfully connect to external SOAP web service.  In the following example, we will try to make SOAP web service calls to web service which requires a certain level of security is met, i.e. web service security. For this purpose we will use a simple "Hello World" web service. We will show you how to leverage WS-Security concept in Orchestra.

By using Orchestra Designer, you will be able to configure following settings.

•Use different actions for SOAP message like timestamp, sign and encrypt actions.

•Select signature and encryption parts of your SOAP message.

•Configure signing and encrypt credentials using  Private/Public key pair in PKCS#12 format.

•Use different signature and digest algorithms

Step Overview

In order to successfully configure security settings for SOAP web service , several steps need to be followed:

1.Configure Security Credential for signing and encrypting SOAP message

2.Create Orchestra Service Declaration by importing web service WSDL file

3.Apply web service security configuration settings for created Service Declaration

4.Incorporate created service declaration to your process model

5.Start process model in order to successfully exchange data with the service

Prerequisites

In order to successfully configure web service security in SOAP web service calls using Orchestra, some basic knowledge about WS-Security concept is required.

Used Concept

Orchestra introduces Service Declaration concept which can be used in order to connect to an external SOAP web service. To be able to sign and encrypt SOAP messages Orchestra Credentials concept is used.

Steps

The following text will give you a detail step-by-step explanation how to apply web service security settings.

Step 1 - Configure Security Credential for signing and encrypting SOAP message

In order to be able to successfully sign and encrypt SOAP messages  two different types of credentials need to be created.

•Right click on the Credentials option from Orchestra Designer main menu.

•Select the option create to create new credentials entry which will be used as signing credentials  for SOAP messages.

•Create Technical login type of credential using Private/Public key pair in PKCS#12 format as authentication option. See image below.

WS Security - create technical login credentials

•Right click on the Credentials option from Orchestra Designer main menu.

•Select the option create to create new credentials entry which will be used as signing credentials  for SOAP messages.

•Create Client login type of credentials using Private/Public key pair in PKCS#12 format as authentication option. See image below.

WS Security - create client login credentials

Previously created credential entries will be used in Service Declaration configuration of the web service we are using in this example.

Step 2 - Create Orchestra Service Declaration by importing web service WSDL file

•Right Click on the Service Declaration option from Orchestra Designer main menu

•Select the option Import WSDL

Service Declaration - Import WSDL option

•After clicking on the Import WSDL option you will be presented with the Import web service description wizard dialog

•Select the WSDL file and click on the Next button

Note: in case you are using WSDL files with nested complex types you can simply type HTTP address of WSDL file in WSDL input field.

•New Screen will be presented to you to adjust additional settings. Click on the Next button to continue.

•After successful importing of the WSDL file you will be presented with the status screen. Click Finish.

WS Security - import WSDL dialog

•Now the Service Declaration is created and configured based on WSDL file definition.

 

Step 3 - Apply web service security configuration settings for created Service Declaration

•Double click on the Service declaration element () from main Orchestra Designer menu.

•On the left side of the opened configuration view you will find Bindings and Operations configurations.

•Double click on the Ports element under the Bindings section to apply additional changes for your Service declaration.

•You will be presented with the configuration panel containing the Web Service Security Settings. See image below.

SOAP Web Service Security configuration

•Select Logging Level value. This value determines on which log level will be SOAP message logged. This is useful if you need to trace SOAP messages that are sent and received.

•Select Sign value for SOAP Message Action. This value is combination of actions performed over SOAP message.

•Signature Parts says which elements of SOAP envelope will be included in signature generation. Signature is generated using  XMLDSig W3C recommendation.

•Encryption Parts says which elements of SOAP envelope will be encrypted. Encrypted data is generated using XMLEnc W3C recommendation.

•For Signing Credentials select Technical Login credentials sec_cred_sign_soap, created in step 2.

Signing Credentials is credential of type Technical login (it has to contain private key of sender - PKCS#12 file) and it is required to sign SOAP message.

•Signature Algorithm is algorithm that is used for signing content (digested Signature Parts).

•Digest Algorithm is algorithm used for cryptographic digest operation over Signature Parts.

•For Encryption Credentials select Client Login credentials sec_cred_crypt_soap, created in step 2.

Encryption Credentials is credential of type Client login (has to contain public key of receiver - public key certificate) that is used to encrypt content (Encryption Parts).

•Select X509 Key Identifier for Key identifier type. Key identifier type is determining format of key transferred in SOAP envelope.

Step 4 - Incorporate created service declaration to your process model

For purpose of testing connection to a external HelloWorld web service we will use following example.

Using the previous created Service Declaration we will fetch the data about weather forecast for specific place name and store it to a xml file.. In order to do this we will incorporate Service Declaration inside existing process model.

•Select your process model from Orchestra main menu.

In order to create new process model go to Process Model reference manual

•Click on the Service Declaration element and place it to your process model diagram.

WS Security - process model example

Note: For this example we will use a simple process model implementation. The model contains File Listener as an inbound channel used as a starting event and File Writer as an outbound channel for storing web service response data. File Listener channel is configured to periodically listen for file changes inside defined file path on the local system. The file contains request data  and  will be used as an input parameter to our Service Declaration. The file can be created based on the message type inside our service declaration which is generate by importing WSDL file.

WS Security - Create xml input file

•Connect the Service Declaration to the service call element

•Configure process model and select appropriate method for calling the "Hello World" web service.

WS Security - service call configuration

•Select the output xml file for storing retrieved data from web service (File Writer channel configuration).

Step 5- Start process model in order to successfully exchange data with the service

Using the process model created in previous step start collecting the data.

•Start test environment to collect the data.

•Place file with request data which can be picked up by the File Listener channel.

•Open the file created by File Reader Channel. File should contain data from web service response.

•Check xml file and verify the data.

Example SOAP message

SOAP message before applying WS-Security looks like:

<SOAP-ENV:Envelope

 xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/"

 xmlns:enc="http://schemas.xmlsoap.org/soap/encoding/"

 xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

 <SOAP-ENV:Header/>

 <SOAP-ENV:Body xmlns:typens="http://sample.security.ws.soffico.de/">

   <typens:sayHi>

     <arg0>Hello World</arg0>

   </typens:sayHi>

 </SOAP-ENV:Body>

</SOAP-ENV:Envelope>
