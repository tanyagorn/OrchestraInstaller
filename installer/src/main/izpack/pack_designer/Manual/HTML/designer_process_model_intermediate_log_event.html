Log event


Topic content

Description

The log event  is an intermediate event which is responsible for logging text information to certain log facilities.

Create

1. Create a process model

2.Then point the mouse on "process", as default, this field is on the right side of the Orchestra window.

3.Click on the folder intermediate events to see its content.

4.Click on "log event".

5. Click on any place in your process model.

Icon

Icon of a log event

Configuration

1.Double click the log event icon.

configure the log event

2.In the configuration menu, you can specify where to write log messages:

a.In the field named Event type you can set the log level

b.If you select the check box Processlog the message is written to the steps of activities executed for the process model. To see that information the log level  for the process model must be set to activity enter/leave or higher.

c.In the field named Log format you specify the format of the text messages written into the Orchestra log, the Scenario log and to the System event log. Actually here you may use the template parameters PROCESS_MODEL, NAME, PROCESS, TOKEN and MESSAGE where PROCESS_MODEL is the name of the process model, NAME is the name of the log event configured in the field Name, PROCESS is is the unique id of the process model and message is replace by the text to log.

d.In the section Syslog sender you can select entries from a list of connection names configured in the syslog client component. The message then will be sent to all of these syslog receivers.
If you select the check box All syslog receivers the message will be sent to all available and activated syslog receivers.
If you select the check box Enqueue Syslog, for each syslog receiver the invocations will be ansynchronously enqueued in a persistent queue. The calling process model can then continue while the syslog send operations will be executed in the background. If then the syslog receiver is not available or the send operation fails the invocations will be retried until the message can be sent.

3.In the tab input assignments you must assign a text expression to the paramter MESSAGE.

 

Sending syslog messages using a scriptlet

An intermediate event doesn't allow to retry an activity nor is it possible to attach an error handling. Therefore there is a second possibility to send syslog messages from a process modell using the scriptlet Syslog message.

Scriptlet to send a Syslog message

The scriptlet has basically four parameters:

Parameters for the scriptlet

1.The parameter logLevel has eight possible values: EMERGENCY, ALERT, CRITICAL, ERROR, WARN, NOTICE, INFO, DEBUG. Normally you should use ERROR, WARN, NOTICE or INFO.

2.The parameter message is a String containing the actual ATNA message (An XML document containing the Audit Log Message as defined in DICOM PS3.15 Annex A.5).

3.The parameter PROCID might be used to group messages belonging together. Typically we add here an invocation of soffico.customerapi.process.ProcessAdapter.obtainGlobalSessionIdentifier(). This method returns a global session id which identifies all activities executed while handling a particular IHE transaction.

4.The Parameter syslogTargetNames may be empty or a comma separated sequence of syslog target names. If the parameter is empty the syslog message is sent to all activated syslog receivers.
