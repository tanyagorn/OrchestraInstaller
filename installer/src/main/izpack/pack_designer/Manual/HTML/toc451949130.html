Book an appointment slots for a patient  (Method POST)


Topic content

 

In this chapter we will create a Orchestra REST Client for requesting an appointment with doctor “rbauer”  and save the booked slot to your File system. According to 3.3.6 Book an appointment slots for a patient  (Method POST) we request the appointmentService by using the URL.

For booking of appointment slots 13 for patient "mpatient" with doctor "rbauer" we use the POST Methode for the URL.

http://localhost:8080/AppointmentMgr/appointments/slots/{slotid}

 

For this Request we have to tell the the server, for which patient we want to book the slot, so we have to set the Bodycontent

<appointmentRequest>

   <patientName>mpatient</patientName>

</appointmentRequest>

of our Request. We also have to inform the server which Content-Type we send in our Bodycontent, so we have to add the header Content-Type: application/xml.

1.Copy your shell scenario from chapter 4.5.1 Preparation.

2.Create a Message Type appointmentRequest for the Bodycontent. Append a child node patientName.

Figure 5 – Book an appointment slot for a patient – Message Type

3.Create a TragetOnly mapping for the Message Type appointmentRequest with an input string variable “patient”.

Figure 6 – Book an appointment slot for a patient – TargetOnly mapping

4.Open your process model.

5.Add the Rest Service Client, the TargetOnly mapping and a call activity to your process model.

6.Link all elements in your modell.

Figure 7 – Book an appointment slot for a patient – Process Modell

7.Now you can configure the Input/Output variable assignment of the TargetOnly Mapping.

Figure 8 – Book an appointment slot for a patient – Input/Output variable assignment TargetOnly Mapping

8.Now you can configure your call activity for the AppointmentMgr-Service. Open the register Service method and select the POST-Method for /appointments/slots/{slotid}.

9.Assign the Response MSG of the call to the CURMSG of the process modell.

 

10.Assign the Header-Parameters ContentType “application/xml”, the Template-Parameter slotid “13” and the Bodycontent MSG (Use the MSG you created in the TargetOnly mapping).
