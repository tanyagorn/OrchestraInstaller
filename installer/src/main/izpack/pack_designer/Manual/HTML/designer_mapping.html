Mapping


Topic content

When dealing with integration of heterogeneous systems a typical task is the conversion of data structures between different target systems. Different systems are normally autonomously developed. Even if such system use the same sort of data (e.g. person, material, patient) it is typical in such environments that a closer look to the data structures reveals that there are differences in structure of the data, data types, semantics and representation of the data. In order to cope with this problem, mappings have to be defined that allow the translation of different data types and formats. Orchestra provides  different methods of dealing with such translations.  

 

The following mapping methods are available within Orchestra:

 

•Graphical mapping: Orchestra provides a graphical mechanism for defining a translation. By means of drag & drop, the user can define the conversion rules in a flexible way

•XSLT mapping: Nowadays, XML-Standards are widely used and know in the context of integration. Since XSLT is a very powerful mechansim, Orchestra allows the use of mappings written in the XSLT language.

•Procedural mapping: Complex mappings may require a lot of logic in order to solve the transformation problem. Since a graphical mapping has a reduced flexibility compared to a programming language Orchestra provide a procedural mapping approach that used a sub dialect of Java to define the transformation.

 

The type of transformation that is used depends on the mapping problem that has to be solved. In general, graphical mappings are a very powerful way. If the mapping becomes to complex, XSLT or a procedure mapping can be used.
