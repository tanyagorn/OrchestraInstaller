Introduction


Topic content

An Orchestra installation can operate in two modes: standalone or cluster. Furthermore it is possible to use the cell system in both operation modes. The cell system is a network of independent Orchestra systems that can be used to invoke service calls and send signals over different Orchestra instances. The cell system is location transparent. Therefore a designer must not care about the fact when designing a scenario.

 

If Orchestra is working in standalone mode, exactly one Orchestra runtime is responsible for execution of the modeled scenarios.

 

When working as a cluster, Orchestra is working in high availability mode. In this setup Orchestra has different physical runtimes that serve together a set of deployed scenarios. The nodes in this group use the same physical database. Furthermore they observe each other in order to detect dead group members. If a dead member is detected the current work is shipped to the remaining nodes that fulfill the open requests.

 

For both operation modes the cell system can be used.
