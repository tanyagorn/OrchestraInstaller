Parser language


Topic content

Description

The parser language is a functional and declarative programming language.

Semantic

The language is functional, because there are no statements, that can change the inner state of the parser, especially there are no allocations. A statment like x=y is a unification, which means it will be checked if the values are the same, if the left variable has no value, it will have the value of the right variable after executing the statement.

The language is declarative, because the stucture of the input text is described and it is not explicitly outlined how the structure will be  processed. The parser rather tries to align the structure with the input. If this is successful, an output document is created.

 

The structure is defined by a hierarchy of definitions. Can this definitions not be aligned, the parser goes back in the input and aligns another definition. All variable bindings will be removed in this step. This process is also known as backtracking.

 

The only statements with side effect are procedure calls. These will always change the output (XML document). The names of the procedures end with an explanation mark.

Syntax

You can find the syntax of the parser language EBNF below.

Script ::= {Definition }

Definition ::= CharacterClassDefinition | ClauseDefinition

CharacterClassDefinition ::= "character" "class" charClassName ":"

CharacterClass { ( "or" | "and" ) CharacterClass }

CharacterClass ::= [ "not" ] ( "[" Range "]" | charClassName )

Range ::= { char "-" char | "\n" | "\r" | "\t" | char }

ClauseDefinition ::= "define" ClauseSignature ":"

Executable { "," Executable } "end"

ClauseSignature ::= name [ "(" ParameterDef { "," ParameterDef } ")" ]

ParameterDef ::= [ TypeName ] parname

TypeName ::= "Value" | "List" | "Atom" | "Boolean" | "String"

| "Number" | "Integer" | "Charset" | "Constructor"

Executable ::= LiteralString | ParameterName | Unify

| charClassName "*" | ProcedureCall

| FunctionCall | Goal | Loop | Try

Unify ::= ParameterName "=" Expression

ProcedureCall ::= procedureName [ "(" Expression { "," Expression } ")" ]

FunctionCall ::= functionName [ "(" Expression { "," Expression } ")" ]

Goal ::= goalName [ "(" Expression { "," Expression } ")" ]

Loop ::= "repeat" [ "greedy" ] [ Expression ] ":"

Executable { "," Executable } "end"

Try ::= "try" Executable { "," Executable } "end"

Expression ::= Literal | ParameterName | FunctionCall

| ListConstructor | charClassName

Literal ::= "[" "]" | LiteralString | LiteralInteger

ParameterName ::= <name starting with upper case>

ListConstructor ::= "list" "(" [ Expression { "," Expression } ] ")"

Syntactical Shortcuts:

•A literal String as a goal is a shortcut for the expect goal. E.g. ";" is a shortcut for expect(";").

•A charClassName followed by a "*" is a shortcut for the invocation of the function readString(space).

•A parametername as a goal is a shortcut for the read goal. E.g. is varname the shortcut for read(varname).
