How to install a Bugfix


Topic content

Structure of a cumulative bugfix file

A cumulative bugfix is a zip file having a name like orc<version>_<date>_cumulative_bugfix_<version2>-build<buildnr>.zip.

An example for the name of a bugfix file is orc4614_20170707_cumulative_bugfix_4.6.1.4-build28.zip.

A cumulative bugfix file always contains

•A File named <filename>.txt containing a description of the fix, e.g. orc4614_20170707_cumulative_bugfix_4.6.1.4-build28.txt

•Subdirectories to be copied into the Orchestra directory orchestra/WEB-INF. So typically a bugfix contains two subdirectory named classes and lib. The subdirectory classes resembles the full structure of the directory  orchestra/WEB-INF/classes. The subdirectory lib contains a set of changed jar files. The changed jar files get a new version number. If a jar file gets a new version number the bugfix contains a dummy file with the old version number which is an empty zip file. This way the user can simply copy the content of the directory over the existing content and replace existing files.
If for example your lib directory contains two files orchestra-domain-4.6.1.0.jar and orchestra-domain-4.6.1.1.jar then the file orchestra-domain-4.6.1.0.jar will be an empty zip file. If we then deliver a new cumulative bugfix it might contain three files orchestra-domain-4.6.1.0.jar, orchestra-domain-4.6.1.1.jar and orchestra-domain-4.6.1.2.jar where orchestra-domain-4.6.1.0.jar and orchestra-domain-4.6.1.1.jar are empty zip files.

To install a bugfix

1.unpack the zip file

2.copy the subdirectories contained in the unpacked bugfix directory to the directory <tomcat>/webapps/orchestra/WEB-INF. Ensure that existing files are replaced.
