State of the cluster nodes


Topic content

For each node in the cluster environment an global state is maintained. (This state can be found in the database table ORC_CLUSTER_ENVIRONMENT).

 

 

"ACTIVE"

The node is running and can be reached by the cluster heart beat

"INACTIVE"

The node is not reachable by the cluster heart beat. A repeat test of the heartbeat didn’t work. Therefore the node is marked as „INACTIVE“ (Reason: e.g. network problems, hardware crash, crash of the Orchestra software)

"SHUTDOWN"

The node is currently shutting down; it was initiated by a user manually.

"SHUTDOWN.OK"

The node is down. The shutdown was initiated by a user.

 

This state affects the behavior of Orchestra:

 

•When all nodes have the state "ACTIVE" a deployment is allowed without any restrictions. Orchestra ensures that all nodes are synchronized after the deployment operation.

•When there is a mixture of states between "ACTIVE" and "SHUTDOWN" or "SHUTDOWN.OK" a deployment is also possible. The nodes in state "SHUTDOWN" or "SHUTDOWN.OK" receive their new configuration during startup.

•If one node is in state "INACTIVE" a deployment is not possible.

 

This is reasonable, since in this situation Orchestra can not guarantee that a deployment will be consistent over all nodes.
