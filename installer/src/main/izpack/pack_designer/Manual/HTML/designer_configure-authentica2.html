Authentication

You can choose between

•basic (user/password) authentication - means that the client has to deliver username and password. For the username an appropriate user credential of type client login is searched and its password is compared with the delivered password. Because with this type of authentication the password is sent in Base64 encoded form and thus can easily be retrieved you normally should use it together with a secure HTTPS connection.

•digest(user/password) authentication - Digest Authentication communicates credentials in an encrypted form by applying a hash function to the username, the password, a server supplied nonce value, the HTTP method, and the requested URI. Thus with this type of authentication the password is not transferred over the wire, so you no  not necessarily need a secure HTTPS connection to secure you authentication.

•certificate authentication: authentication using a X509 certificate.

 

Authorization using roles

Configuring authorization

The user authorization is done using roles. In every case the found user credential must have a role which as configured in the list of allowed roles in the field named Authorization. Otherwise Orchestra responses with a HTTP 403.
