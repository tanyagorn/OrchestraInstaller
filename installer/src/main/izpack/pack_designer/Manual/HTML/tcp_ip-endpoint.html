TCP/IP Endpoint


Topic content

Description

Environment entries of type TCP/IP endpoint are used to declare the parameters of a TCP/IP communication point. The entry is used by Channels, waiting for TCP-connections.

Purpose of object

It is used for inbound channels , e.g. MLLP Receiver.

Configuration

Step by Step

1.Create an Environment entry of type TCP/IP endpoint.

2.Fill in the port to access the endpoint.

3.Fill in the number of Simultaneous Connections.

4.Fill in the Connect timeout: How long should the connection process be at most.

5.Fill in Max idle time: How long should the connection be running without any process.

6.Select the intended protocol:

oTCP: Core protocol of the Internet protocol suite.

oTLS(different versions): Cryptographic protocol to provide communication security.

oSSL: Predecessor of TLS.

7.Security Settings:

oDecide whether to use Authentication.

oIf authentication is selected, choose a Trusted signer to connect with. Therefore create Credentials of type trusted server and add authentication of type public key pair.

8. A test of TCP/IP endpoint isn't possible, so click Finish.
