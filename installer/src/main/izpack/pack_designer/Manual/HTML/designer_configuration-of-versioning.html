Configuration

In order to activate this versioning support, it has to be enabled by manual editing the following file "soffico\config\repository_type_description.xml". Within that file one has to uncomment the entry for the required versioning system. As an example, the following snippet shows the activation of the svn-support:

<repository_type_description>

 <!-- Configuration properties for repository types -->

 

 <repository_type_descriptor                repositoryType                        = "SVN"

                                                 name                                        = "Subversion"

                                                 configDirectoryName                = ".svn"

                                                 description                                = "Subversion implementation"

                                                 repositoryImplClass                = "soffico.versioning.impl.svn.SvnRepository"

                                                 repositoryLoaderClass        = "soffico.versioning.impl.svn.SvnRepositoryLoader"

                                                 repositoryGuiClass                = ""

                                                 transferTypes                        = "svn, file, http, https">

 </repository_type_descriptor>

</repository_type_description>

 

Adjusting the designer settings

When subversion support is used, the following parameters have to be added to the command line of the scenario designer.

-Dsvnkit.http.methods=Basic,Digest,NTLM

 

Note: This settings are adjusted in "designer.cmd". In the following you can the the content of this file with active subversion settings

REM @echo off

setlocal

title OrchestraDesigner

 

call "%~pd0orchestra_env.bat"

 

@set CP=%ORC_HOME_WEBINF%\classes;%ORC_HOME_WEBINF%\lib\*;%ORC_HOME%\lib_designer\*

 

"%ORC_JAVA_EXEC%" -showversion -Xmx512M -Dsvnkit.http.methods=Basic,Digest,NTLM -cp "%CP%" emds.epi.graph.EPIEditor %*

 

endlocal
