Java Functions for Process Models


Topic content

Java functions used in Process models for variable assignment (Properties, Process Variables, Parameters)

Common static methods

Some of the available methods exist only for historical reasons. We have to support them further to ensure that existing scenario will work. Methods we consider to be obsolete are marked with the word @deprecated

The following two methods do basically the same. There are two variants for historical reasons.

@param message of which the root node should be evaluated

Object evalXPath( Message message, String expression, BaseType targetType, String namespaceDecls ) or        

Object xpath( String expression, String targetType, ExpressionContext context) or

Object xpath( Object baseObject, String expression, String targetType, ExpressionContext context )

Evaluate an XPath expression against a structured Message.

@param message/baseObject:  The message on which the expression is evaluated. also the parameter declared to be an object MUST be a Message, otherwise an Exception is thrown.

@param expression XPath expression text

@param targetType the expected result type e.g. SimpleTypes.STRING or ExtendedTypes.NODE_SET

@param namespaceDecls optional namespace declarations e.g. "tns='http://my.com/sysman/parinf/v1' myns='emds:mynames'"

@param context context from which the namespace declarations and variable declarations come

@return the result of the evaluation; the type of the result is the associated class of targetType        

Hint: If you want to access a structured message using XPath it is preferable to select the expression language XPath. Then use also can use the XPathEditor. to create the XPath expression.

If you really have to use it you may get an the expression context referencing the implicit variable _CTXT_ .

E.g. assume you have a Variable HEADER of type Message and the DOM tree in this message has a default namespace "urn:emds:mail". Now you want to select the XPath "mail:Subject" on the root node of that message. Yo may

•simply select XPath as Expression language and write

•or you may use Java

There is almost never a reason to prefer Java instead of XPath.
