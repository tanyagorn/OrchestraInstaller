Mapping archive


Topic content

Description

oA mapping archive is a standard java archive that contains user defined mappings on basis of XSLT or Java

 

User defined java mapping:

A user defined java mapping is useful when the scenario designer wants to implement the mapping completely by its own. In order to define a java mapping on has to extend the base class "ExecuteableDOMMapping" defined in the orchestra customer API (orchestra-customer-api-x.x.x.x.jar). This method is defined as follows:

 /**

  * This function is the main mapping function that has to be implemented individually for every java mapping.

  * Every user defined mapping must overwrite this method. The framework provides the DOM-Model of the source message,

  * the DOM-Model of the target messages and additional message properties. Note: the latter two parameters must be used

  * for passing back the result to orchestra. When the method returns, the current state of the target message and properties

  * is taken to create the orchestra message.

  *

  * @param sourceMessage                Representation of the source message as simple DOM-Tree

  * @param targetMessage                Initial DOM-Document that must be used to construct the target message. Add new elements only to this document.

  * @param properties                Additional properties that should be added to the orchestra message

  */

 

 public abstract void transformMessage( Document sourceDocument, Document targetDocument, Map<String,String> msgProps );

 

In the following you will find a example for a simple mapping that just copies the source document to the target document.

package soffico.customerapi.mapping;

 

import java.util.Map;

 

import org.w3c.dom.Document;

import org.w3c.dom.Node;

 

/**

* Simple mapping that demonstrates how user defined java mappings

* can be implemented

*/

 

public class SimpleCopyMapping extends ExecutableDOMMapping {

 

 /**

  * {@inheritDoc}. Copy only the source document to the target document

  */

 

 @Override

 public void transformMessage(Document sourceDocument, Document targetDocument, Map<String, String> msgProps) {

 

         /*

          * Create a copy of the root element of the source document

          */

         

         Node copyRoot = targetDocument.importNode( sourceDocument.getDocumentElement(), true );

         

         /*

          * Add this copy the the target message

          */

         

         targetDocument.appendChild( copyRoot );

         

         /*

          * Append some properties to the orchestra messages

          */

         

         msgProps.put( "MESSAGEATTRIBUTE", "Created by SimpleCopyMapping" );

 }

}

 

Note: This kind of mapping completely operates on the Document Object Modell (DOM). All operations are defined in the java api (org.w3c.dom.*) can be used.

Since Orchestra 4.6 it is also necessary to include orchestra-domain-x.jar and orchestra-commons-x.jar in the Development Environment to find dependent classes (like EpiBaseException).

 

XSLT mapping:

A user defined XSLT mapping is useful when the scenario designer wants to implement the mapping completely on basis of the Extensible Stylesheet and Transformation language, whereby Orchestra currently supports XSLT 1.0. Normally the mechanism of an mapping archive must not be used when working with XSLT because Orchestra allows the direct edition of XSLT-Mapping in the orchestra designer.
