Operations


Topic content

Description

Service declaration implies declaration of operations (methods) that a service can perform. Simply said, operations describe capabilities of a service and they are crucial for service declaration.

In case you created Service Declaration manually or you simply want to modify service, you can use operations section in Designer that allows operation editing.

Creation

In existing service declaration please navigate to ServiceDeclaration->Operations branch in tree that is contained in service declaration panel. Right click on Operations item in the tree and select Create operation:

Creating an operation from context menu

New child item name will be added at the end of the tree. If you click on the item, details about operation will be shown in the panel:

Operation name

Edit operation name in the Operation field and save the current element (CTRL+S).

Configuration

After successful creation of operation we have to configure it, i.e. set input and output parameters.

You can also set Description for the operation.

Exchange Pattern

This configuration is used to denote method behavior and it is similar to Message Exchange Pattern (MEP) from web services. There are following types of exchange patterns:

- Not defined

- Request - operation receives a message but does not return a response;

- Request/Response - operation receives a message and returns response;

- Notify - operation sends a message and does not expect response;

- Solicit/Response - operations sends a message and waits for response.

Default exchange pattern is Request/Response

Parameters

Defining input and output parameters of the operations is done in Parameters table. Simply click on table cell to edit the parameters:

Parameter assignment

Please have in mind that you need to set Usage - direction of the parameter.
