Orchestra SSL - truststore configuration

Orchestra contains trust-store that is used for storing public certificates of Certification Authorities (and chains) that are considered as trusted. All certificates that are issued by certificates contained in trust-store will be successfully authenticated to orchestra during SSL Handshake process.

The following basic parameters are preset:

•TRUSTSTORE:                Trust.store file

•TRUSTSTORE.PASSWORD:        The passwords for the trust-store access are deposited in the Orchestra configuration. All passwords are saved encrypted. The encryption is done with                                the method "PBEWithMD5AndDES"

•TRUSTSTORE.TYPE:                Java trust-store, all common trust-store formats like JKS or PKCS12 are supported

The configuration for the usage of the trust-store is done in the file orchestra_configuration.xml. The following parameters within the group SSLSettings can be set.

Parameter

Description

TRUSTSTORE

Location of the trust-store, this has to be inside the classpath of Orchestra, the default value is config/orchestra_keystore

TRUSTSTORE.PASSWORD

Password for accessing the keystore, encrypted according to "Orchestra password encryption"

TRUSTSTORE.TYPE

Type of the used keystore, possible values are „JKS“ (Java keystore) and „PKCS12“ (Keystore in PKCS12 format)

 

Trust-store has to be enabled in orchestra_configuration.xml!

 

Trust-store usage is not enabled by default. You will have to set additional parameter in orchestra_configuration.xml in ServletEngine group:

Parameter

Description

SSLPORT.USE.TRUSTSTORE

Enforcing trust-store use in SSL Handshake, when orchestra acts as a server. When this parameter is set to false (default), all certificates will be considered as trusted

 

Trust-store has to be in the config directory of Orchestra! It is not possible to use an absolute path outside Orchestra!
