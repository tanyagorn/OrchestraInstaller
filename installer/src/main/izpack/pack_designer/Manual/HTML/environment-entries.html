Environment entries


Topic content

Concept

Environment entries define the location of input and output resources. They contain the specific information about how to access the resource. If a Scenario is deployed in the Orchestra runtime,  its Environment entries can be modified to reference the specific resources in that environment. Even if a Scenario is redeployed, the modified Properties of Environment entries are not changed. This way a developer can improve or change a Scenario  in the designer, then redeploy it in an integration system to test it there and finally redeploy it on the production system without need to adapt the environment entries each time again and again.

Icon

Process-model integration

Environment entries aren't directly used in process model. The define the start or end point of channels and appear in process model indirectly via Channels.

Creation

oTo create a new entry right click on folder environment entries in the left hand tree and select create.

oSelect a name and an optional description for the entry and click on Next. Possible name convention: ee_<type of entry>_<in/out>

Configuration

To reconfigure the settings double click on the entry in the left hand tree or select right click>properties. Change the intended information in the appearing window.
