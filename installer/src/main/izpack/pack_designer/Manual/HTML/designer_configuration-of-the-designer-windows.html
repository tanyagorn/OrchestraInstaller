Configuration of the designer environment


Topic content

Before the designer and other command line tools can be used, the environment of the orchestra designer has to be adjusted. This can be done in the file "startscripts\orchestra_env.bat" (or orchestra_env.sh for Linux)

 

rem --------  adjust these values
set ORC_JAVA_HOME=C:\Program Files\Java\jre8
set ORC_HOME=C:\Program Files\Apache Software Foundation\Tomcat 8.5\webapps\orchestra
rem --------
 
set ORC_JAVA_EXEC=%ORC_JAVA_HOME%\bin\java
set ORC_HOME_WEBINF=%ORC_HOME%\WEB-INF

 

 

Setting

Usage

Default

Possible values

ORC_HOME

Overrides the default value if not empty. Defines the folder of where orchestra is located (sub-folders of this folder are themes, WEB-INF, etc.) .

Empty, uses a relative path "./orchestra"

Any Path, relative or absolute on the file system. E. g.

ORC_HOME="C:\Orchestra Designer\4.5.6.2\orchestra"

Don't forget the double quotation marks at least when you have spaces in the path.

ORC_JAVA_HOME

Overrides the default value if not empty. Defines the path to the Java folder of the Java version for orchestra.

Empty, automatically retrieves Java version from sub-directory "Java" or afterward's from Windows registry

Any Path, relative or absolute on the file system. E. g. "C:\Program Files\Java\jre7"

Don't forget the double quotation marks at least when you have spaces in the path.

 

 

designer.cmd (or designer.sh)

Parameter

Description

-Xmx256M

Sometimes it is necessary to start the Orchestra designer with more RAM. So adjust this value.

-Djava.net.preferIPv4Stack=true

Prefers IPV4 Communication over IPV6. Speeds up test engine startup in some environments where IPV6 is not rolled out but enabled on the machine itself

nolock

Ignore existence of Lock-file; Not recommended in environments where more than one user has access to the scenarios open
