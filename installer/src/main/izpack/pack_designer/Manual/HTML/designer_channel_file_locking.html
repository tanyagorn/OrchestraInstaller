File locking


Topic content

Concept

A channels File Listener, File Reader and File Writer may lock the file before processing to avoid that the file content is written or otherwise changed by another program while it is processed by the channel.

If the file cannot be locked exclusively because it is locked by another file, the channel operation is canceled. In the context of the File Listener this means that it is simply tried again next time (after the polling period). In the context of the File Writer or the File Reader the Channel tries several times to the get the lock (the actual number is configurable) and between that waits for a short time, typically 200, then 400 then 800 milliseconds (this is also configurable). If it cannot acquire the lock after these tries the channel operation fails.

There are several locking methods available depending on the selected file system type (local, SMB or NFS).

Creation

A lock method is always selected and configured in the course of configuring a Stream channel.
